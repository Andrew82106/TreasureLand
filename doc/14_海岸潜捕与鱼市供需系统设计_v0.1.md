## 1. 设计结论

旧有浅滩采集是“交互一次，随机获得40—80金贝”的保底收入点，缺少操作、物种、选择和世界反馈。海岸潜捕将其升级为一条完整但可控的生活经济循环：

```text
阅读海况、鱼群与收购行情
        ↓
选择潜捕地点与装备
        ↓
在氧气和鱼篓容量内抓鱼
        ↓
携带鱼获上岸
        ↓
立即出售 / 完成订单 / 留作收藏 / 短期保存
        ↓
玩家与NPC供给进入鱼市，后续价格、新闻和产业经营发生变化
```

模块承担四项职责：

1. 为贫穷状态提供无本金、有限且可理解的恢复路线；
2. 用氧气、距离、鱼篓和鱼类行为形成轻量操作选择；
3. 让天气、时段、节庆、NPC与商店共同形成可解释的鱼价；
4. 为后续蓝鳍渔业份契提供真实经营数据和生活中的市场观察入口。

正式模块名使用“海岸潜捕”，地图交互点使用“海岸潜捕点”，行动按钮使用“下水抓鱼”。正文不使用“海边摸鱼”作为系统名称。

## 2. 体验目标与非目标

### 2.1 体验目标

- 玩家下水前能根据天气、时段和价格选择目标鱼，而不是只追逐稀有度。
- 玩家下水后持续面对“继续深入还是带着现有鱼获上浮”的轻量风险判断。
- 普通鱼提供稳定收入，稀有鱼提供可预期但不保证出现的小型财富惊喜。
- 鱼价变化有供给、需求、库存或事件原因，界面能够解释主要驱动。
- 玩家抓到的鱼会进入商店库存、居民订单、新闻和蓝鳍渔业经营，而不是凭空换钱后消失。
- 氧气制造节奏和取舍，但不把明亮海岛生活变成死亡惩罚或高压生存。

### 2.2 非目标

- 不制作复杂潜水生存模拟，不加入死亡、潜水病、装备耐久和繁琐补给。
- 不把抓鱼做成水下战斗，不要求攻击、伤害或击杀鱼类。
- 不让稀有度直接等于固定高价；市场需求不足时，稀有鱼也可能暂时卖不出最高价。
- 不让几十种鱼各自进行无法理解的独立随机涨跌。
- 不允许玩家通过无限下水、读档重掷或同价批量倾销破坏基础经济。
- 不把有限鱼获混入永久万物图鉴，不改变造化盆固定二元配方规则。

## 3. 核心流程

### 3.1 下水前

玩家在海岸潜捕点查看：

- 当前时段与天气；
- 三个潜捕区域的能见度、洋流和常见鱼群；
- 已装备的氧气上限、移动能力和鱼篓容量；
- 蓝鳍鱼铺当前重点收购鱼种、报价与剩余需求量；
- 已接受的餐厅、居民或收藏家订单；
- 本日剩余有效鱼群窗口。

玩家可以先去鱼铺看价，也可以直接下水。公开行情免费查看；看价界面进入`ui_paused`且不追加固定潮刻。

### 3.2 水下阶段

一次潜捕由真实时间内的短场景构成。进入时切换为`activity_snapshot`，世界自然时钟停止，氧气继续使用玩法自己的现实秒数：

1. 玩家从区域入口入水，氧气开始下降；
2. 玩家观察鱼影、路线、深度与剩余氧气；
3. 靠近目标鱼后使用抓取键；
4. 成功捕获的鱼立即占用一个鱼篓槽；
5. 鱼篓已满时可以上浮，也可以放走一条鱼后继续；
6. 玩家主动上浮或氧气归零后结束本次潜捕。

抓取失败只消耗本次活动内的现实时间与少量氧气，不额外扣除金贝，也不会在整次潜捕固定1潮刻之外追加时间成本。

### 3.3 上岸阶段

上岸后统一展示：

- 捕获鱼种、尺寸、品质和当前新鲜度；
- 当前鱼铺估价与订单匹配情况；
- 首次发现、尺寸纪录和海生图鉴进度；
- 本次预计鱼获价值区间；
- 本日剩余有效鱼群窗口。

鱼获先进入独立鱼获箱，不强制立刻出售。完成上岸结算后退出`activity_snapshot`并固定推进1潮刻，不叠加水下实际游玩时长。

## 4. 氧气、鱼篓与失败边界

### 4.1 基础参数

首版建议参数：

| 参数 | 初始值 | 设计作用 |
|---|---:|---|
| 基础氧气 | 50秒 | 支持一次短而完整的潜捕 |
| 低氧提醒 | 剩余25% | 给玩家留下主动返程时间 |
| 临界提醒 | 剩余10% | 强化视听反馈，不隐藏风险 |
| 鱼篓容量 | 4条 | 限制一次收益并迫使玩家选鱼 |
| 有效鱼群窗口 | 每日3次 | 维持现有保底经济上限 |
| 单次时间成本 | 1潮刻 | 与时间、市场和日程统一结算 |

数值为首版调试起点，不是最终手感结论。

### 4.2 氧气消耗

- 正常游动持续消耗氧气；
- 快速游动提高移动速度，同时提高氧气消耗；
- 抓取动作消耗少量固定氧气，失败同样消耗；
- 洋流可以提高或降低特定方向的移动成本；
- 打开暂停、说明或无障碍设置时停止氧气计时。

氧气只限制本次探索范围，不作为跨活动通用体力。

### 4.3 氧气归零

氧气归零时：

1. 立即停止抓取与深入移动；
2. 播放简短的自动上浮演出；
3. 保留已经放入鱼篓的鱼；
4. 正常消耗本次1潮刻和一个有效鱼群窗口；
5. 不造成死亡、金贝罚款、永久属性损失或装备损坏。

主动上浮应当比氧气归零更从容，但不通过额外惩罚制造挫败。

### 4.4 每日次数的世界解释

每日3次不是人物体力，而是漂流湾在潮汐中形成的三个有效鱼群窗口。窗口用尽后仍可查看海面、鱼市和历史记录，但无法继续生成有收益的潜捕场景。次日清晨随鱼群和海况一起刷新。

后续新区域可以增加不同的潜捕入口，但不能无条件叠加为无限保底收入。新增入口应占用同一日程、具有独立风险或需要明确资格。

## 5. 潜捕区域

首版使用一个地图入口、三个水下区域。玩家进入潜捕界面后选择区域，避免在大地图上堆叠三个过近的交互标记。

| 区域 | 深度与环境 | 主要鱼群 | 风险与收益 |
|---|---|---|---|
| 白沙浅湾 | 距离短、能见度高、洋流弱 | 普通食用鱼、群游鱼 | 稳定保底，单位价值较低 |
| 珊瑚礁棚 | 遮挡较多、路线分叉 | 礁石鱼、精品食用鱼、观赏鱼 | 需要观察藏身点，价值中等 |
| 沉船外缘 | 距离远、局部暗流、返程更长 | 夜行鱼、稀有鱼、大型鱼 | 氧气压力最高，出现高价目标 |

区域开放建议：

- 白沙浅湾开局开放；
- 珊瑚礁棚在完成首次潜捕或获得基础海况知识后开放；
- 沉船外缘在获得更好的观察或呼吸装备后开放；
- 区域资格永久保留，不随财富下降关闭。

## 6. 鱼种与行为

### 6.1 鱼种数据

```yaml
species_id: silverfin
name: 银鳍鱼
rarity: common
habitats: [sand_shallows, coral_shelf]
active_periods: [morning, day]
weather_affinity: [sunny]
behavior: schooling
base_value: 14
demand_tags: [daily_food, restaurant]
size_range: [small, large]
shelf_life_days: 2
```

每种鱼至少定义：

- 稀有度；
- 栖息区域；
- 活跃时段；
- 天气偏好；
- 行为类型；
- 基础价值；
- 市场需求标签；
- 尺寸范围；
- 新鲜度期限。

### 6.2 行为家族

首版只制作三种行为家族，通过参数产生物种差异：

| 行为 | 表现 | 玩家判断 |
|---|---|---|
| 群游 | 多条沿固定路线移动，受惊后整体转向 | 容易找到，但连续抓取会让鱼群散开 |
| 藏礁 | 在珊瑚、海草或沉船边缘停留，靠近后短距离逃窜 | 需要观察藏身点和预判出口 |
| 疾游 | 单独出现，周期性加速并改变方向 | 价值较高，但追逐会明显消耗氧气 |

不为每种鱼制作完全独立的AI。物种差异主要来自速度、警觉距离、转向频率、停留时间和活动区域。

### 6.4 尺寸与品质

尺寸系数建议：

| 尺寸 | 售价系数 |
|---|---:|
| 小型 | 0.85 |
| 标准 | 1.00 |
| 大型 | 1.25 |
| 纪录级 | 1.60 |

纪录级鱼首次捕获时进入海生图鉴纪录，并可能触发收藏家或晨报事件。尺寸由潜捕场景生成时锁定，读档不得重新随机。

## 7. 鱼获、保存与海生图鉴

### 7.1 独立鱼获

鱼获是有数量、可出售、会失鲜的商品，与永久万物严格分离：

| 类型 | 是否有数量 | 是否可出售 | 是否可放入造化盆 | 是否永久保留 |
|---|---:|---:|---:|---:|
| 永久万物 | 否 | 否 | 是 | 是 |
| 鱼获 | 是 | 是 | 否 | 否 |
| 海生图鉴记录 | 否 | 否 | 否 | 是 |

鱼获不写入`discovered`，也不能通过商店购买来解锁四阶谱系万物。

### 7.2 鱼获实例

```yaml
catch_id: catch_day3_0042
species_id: silverfin
size: large
caught_day: 3
caught_tide: 6
freshness_state: fresh
source_area: coral_shelf
```

界面可以按鱼种和品质合并显示，但结算必须保留尺寸与捕获时间，避免不同售价的鱼被错误堆叠。

### 7.3 新鲜度

首版建议：

| 状态 | 条件 | 售价系数 |
|---|---|---:|
| 鲜活 | 捕获当日 | 1.00 |
| 尚鲜 | 次日 | 0.65 |
| 加工级 | 第三日起 | 0.40 |

鱼不会因过期直接消失，以免形成隐性惩罚；第三日起只按加工原料价格收购。后续冷藏设备可以延长鲜活或尚鲜状态，但不能永久冻结全部鱼获。

### 7.4 海生图鉴

海生图鉴独立记录：

- 是否见过与是否抓到；
- 常见区域、时段和天气；
- 最大尺寸纪录；
- 已知需求标签；
- 相关NPC、节庆与订单；
- 玩家亲自验证的市场观察。

未发现鱼种只显示轮廓和可能的目击线索。海生图鉴不计入万象塔72种永久万物进度。

## 8. 鱼市供需与动态价格

### 8.1 交易地点

鱼获由椰影街的“蓝鳍鱼铺收购柜台”收购。该柜台是独立交易业务，不并入椰影杂货铺的研究服务页面：

- 椰影杂货铺继续只出售配方方向线索与实验折扣；
- 蓝鳍鱼铺负责鱼价、库存、订单和出售；
- 两者可以位于同一商业区域，但不共用商品目录和交易规则。

查看鱼价与出售鱼获进入`ui_paused`，不追加固定潮刻；玩家比较价格和批量选择的速度不改变报价。

### 8.2 供给

每种鱼的有效供给由以下部分组成：

```text
有效供给 = 鱼铺现有库存 + NPC本时段到货 + 玩家出售量
```

NPC到货受到以下因素影响：

- 天气与洋流；
- 鱼群丰度；
- 蓝鳍渔业船队状态；
- 港口是否正常运转；
- 上一时段价格是否吸引更多渔民供货；
- 节庆或事故事件。

玩家出售应进入真实库存，不能只增加金贝而不改变市场状态。

### 8.3 需求

需求由多个消费方共同生成：

- 居民日常食用；
- 海鲜餐厅采购；
- 游客和节庆需求；
- 收藏家与名流订单；
- NPC委托；
- 蓝鳍渔业加工需求。

鱼种通过需求标签共享需求事件。例如“海鲜节”提高所有节庆鱼和餐厅鱼的需求，而不是给12种鱼分别随机加价。

### 8.4 内部定价

系统内部使用以下基础关系：

```text
供需系数 = clamp(sqrt((有效需求 + 1) / (有效供给 + 1)), 0.60, 2.20)
目标价 = 基础价 × 供需系数
本时段报价 = 上时段报价 × 0.60 + 目标价 × 0.40
单条售价 = 本时段报价 × 尺寸系数 × 新鲜度系数
```

设计目的：

- 平方根抑制小样本造成的极端波动；
- 0.60—2.20的边界避免普通鱼突然归零或翻十倍；
- 价格平滑保留趋势，使玩家能够观察和判断；
- 尺寸与新鲜度只修正具体鱼获，不改变整个物种行情。

最终结算四舍五入为整数金贝，且任何可出售鱼获的单价不得低于1金贝。

### 8.5 时段刷新

鱼价在清晨、白天、傍晚和夜晚的时段边界统一更新，不每秒跳动：

1. 结算上一时段居民、餐厅和加工消费；
2. 生成NPC新到货；
3. 加入玩家上一时段出售形成的库存；
4. 应用天气、节庆、港口和人物事件；
5. 计算目标价与平滑报价；
6. 生成可公开的涨跌原因。

玩家下水前看到的是当前报价；潜捕结束固定推进1潮刻，如果跨过时段边界，上岸后的实际报价可能变化。自然闲逛、站立、快速前往或主动等待同样可能跨过报价刷新点。界面应显示刷新倒计时，并在下水确认前提示这一风险。

### 8.6 收购深度与批量出售

每种鱼显示“当前重点需求量”。在需求量内按完整报价收购，超出部分进入普通加工渠道：

| 出售区间 | 结算方式 |
|---|---|
| 重点需求量以内 | 当前完整报价 |
| 超出重点需求量的第一档 | 当前报价的85% |
| 大量超额部分 | 基础价的60%，再计算尺寸与新鲜度 |

出售确认必须预览每一档数量和总金额。这样玩家大量倾销会得到较低的边际价格，不会以界面顶部的一条报价无限成交。

### 8.7 价格解释

鱼铺不向玩家展示公式，而显示最多三条主要原因：

```text
银鳍鱼　当前32金贝　↑
海鲜节采购增加：+24%
强风导致渔船到货减少：+18%
今日玩家与渔民到货较多：-9%
重点需求剩余：5条
```

同一因素不能同时以不同名称重复计算。传闻可以不准确，但柜台显示的已经发生的库存、成交和公开订单必须真实。

## 9. 跨模块联动

### 9.1 时间、天气与日程

- 清晨生成当日海况、基础鱼群、NPC供给和公开需求；
- 时段决定鱼种活动与价格刷新；
- 天气同时影响潜捕难度、NPC捕获量和市场供应；
- 一次潜捕使用`activity_snapshot`并固定消耗1潮刻；查看行情、整理鱼获和出售使用`ui_paused`且不追加固定潮刻；
- 海鲜节、宴会和风暴作为统一世界事件进入日程，而不是鱼市私有随机效果。

### 9.2 商店与交易

- 蓝鳍鱼铺提供报价、重点需求、库存解释和批量出售；
- 餐厅、居民和收藏家订单提供高于普通报价的定向需求；
- 订单必须标明鱼种或标签、数量、品质、新鲜度、截止时间和奖励；
- 出售与订单交付采用原子结算，不能扣除鱼获后发放失败；
- 鱼铺不出售永久万物、配方答案或实验折扣。

### 9.3 蓝鳍渔业与商会份契

后续商会系统读取鱼市的经营汇总：

```text
蓝鳍渔业收入 = 批发销售 + 餐厅销售 + 加工品销售
蓝鳍渔业成本 = 船队捕捞 + 玩家与NPC采购 + 加工与运输
经营利润 = 收入 - 成本 - 事故损失
```

鱼价上涨不自动等于份契利好：

- 若原因是鱼群稀少，捕捞部门可能减产，餐厅采购成本也会上升；
- 若原因是节庆需求增长且库存充足，销量和利润可能同步增长；
- 玩家卖鱼能补充供给，但开局少量鱼获不足以独自操纵整家商会；
- 玩家成为大户、供应商或主要股东后，才逐步获得显著产业影响力。

首版没有商会份契实装时，鱼市仍记录收入、采购、库存和事件摘要，作为后续接口与晨报素材。

### 9.4 《晴潮晨报》

晨报提供：

- 当日天气与海况；
- 可能出现的鱼群；
- 餐厅、节庆和游客需求；
- 昨日主要鱼价变化；
- 蓝鳍渔业公开经营消息；
- 稀有鱼目击与来源明确的传闻。

晨报只提供公开信息。更精确的栖息位置、数量和订单变化可以来自现场观察、NPC关系或玩家自己的历史记录。

### 9.5 NPC、关系与委托

- 渔民提供鱼群位置、海况经验和竞争性供给消息；
- 厨师发布食用鱼和精品鱼订单；
- 收藏家收购纪录级或观赏鱼；
- 米娅报道异常鱼群、价格和玩家纪录；
- 阿拓可以解释保存工具或万物知识，但不负责收购鱼获；
- 牌桌上的渔业人物可能谈论经营和需求，其观点带有立场，不构成必赚答案。

关系提升提高信息具体度或开放私人订单，不直接增加捕获概率和鱼价。

### 9.6 万物合成

永久万物不被当作消耗品。已发现万物可以提供潜捕知识或解锁服务：

| 永久万物 | 潜捕联动方向 |
|---|---|
| 河流 | 提前解释洋流方向和移动成本 |
| 雨 | 提高天气相关鱼群预测的具体度 |
| 水罐 | 解锁鱼获保存或供气设备的研究服务 |

这些用途只能改善信息、装备或保存能力，不能改变固定配方，也不能在潜捕中消耗万物。

### 9.7 地图与探索

- 漂流湾保留一个海岸潜捕入口；
- 水下三个区域在潜捕界面中选择；
- 首次发现区域、特殊鱼群和沉船线索写入地图；
- 后续外岛潜捕点可以拥有不同生态和市场，但必须纳入统一时间与经济约束。

### 9.8 收藏、住宅与财富成长

- 首次抓到鱼和最大尺寸进入海生图鉴；
- 纪录级与观赏鱼可以进入住宅水族箱或收藏陈列；
- 装备升级包括更好的呼吸设备、脚蹼、鱼篓和冷藏鱼箱；
- 装备属于长期资产，不进入永久万物图鉴；
- 高级装备提高选择权与稳定性，不无限增加每日有效鱼群窗口。

### 9.9 牌会、竞速与节庆

潜捕不直接改变牌序或逐风赛结果，只通过世界事件形成间接联系：

- 海鲜节同时提高鱼类需求、游客数量和金潮俱乐部客流；
- 强风减少渔业供给，同时改变逐风赛场条件；
- 渔业商人可能在牌桌上透露观点或因经营亏损改变入桌资金；
- 玩家供应宴会鱼获可以解锁特殊牌局、赛事贵宾席或人物邀请。

## 10. 装备成长

### 10.1 装备槽

首版预留四类装备：

| 装备 | 主要作用 | 禁止作用 |
|---|---|---|
| 呼吸设备 | 增加单次氧气或降低消耗 | 不增加每日潜捕次数 |
| 脚蹼 | 提高移动效率、减轻逆流成本 | 不自动追踪稀有鱼 |
| 鱼篓 | 增加单次携带量 | 不取消市场收购深度 |
| 保存箱 | 延长新鲜度 | 不永久冻结全部鱼获 |

### 10.2 成长原则

- 开局装备足以完成白沙浅湾并获得稳定收入；
- 第一轮升级应改善手感，不要求玩家先富裕才能使用潜捕；
- 深水区资格可以同时读取装备和一次明确的引导事件；
- 高级装备提高单位时间效率，但潜捕仍应明显低于高风险财富玩法的爆发上限；
- 装备购买与升级进入财富轨迹，卖回时不得无损返还全部价格。

## 11. 经济平衡

### 11.1 收益目标

| 情况 | 单次潜捕目标收益 |
|---|---:|
| 普通新手表现 | 40—55金贝 |
| 熟练稳定表现 | 55—75金贝 |
| 偶遇少见鱼 | 75—120金贝 |
| 单条稀有鱼惊喜 | 120—250金贝 |

每日三次普通潜捕的总收益应能支持多次基础实验和一次低额风险活动，但不能稳定超过牌会、竞速或后续份契系统的合理盈利上限。

### 11.2 防刷与防套利

- 每日有效鱼群窗口限制总产出；
- 氧气和鱼篓限制单次产出；
- 同种鱼大量出售降低边际收购价；
- 新鲜度限制无成本无限囤货；
- 同一存档的鱼群、尺寸和路径由已保存种子决定；
- 读取存档不重掷当前潜捕场景；
- 鱼铺不能把刚收购的鱼原价卖回玩家，不形成无风险往返交易；
- 市场事件必须有持续时间和来源，不允许隐藏的“专门压制玩家”修正。

### 11.3 市场影响尺度

开局玩家是小型个人供货者：

- 能明显影响单个柜台的短期需求量；
- 能轻微影响某鱼种下一时段的库存和报价；
- 不能仅靠三次潜捕决定全岛鱼价或蓝鳍渔业份契走势。

玩家后续取得大鱼篓、外岛航线、餐厅合同或商会身份后，市场影响才逐步扩大。

## 12. 界面与反馈

### 12.1 下水前界面

- 左侧：三个潜捕区域与生态预览；
- 中央：天气、时段、能见度、洋流与可能鱼群；
- 右侧：氧气、鱼篓、装备、重点订单与当前高价鱼；
- 底部：本日剩余窗口、时间成本和“下水抓鱼”按钮。

### 12.2 水下HUD

- 氧气条；
- 当前鱼篓槽位；
- 已抓鱼种与尺寸简标；
- 返回岸边方向；
- 低氧和鱼篓已满提醒；
- 抓取距离与短暂失败反馈。

稀有鱼不使用全屏闪光提前泄露，只通过轮廓、颜色、行为和发现音效建立辨识度。

### 12.3 上岸结算

结算顺序使用“鱼篓展开—逐条识别—价值与订单提示—纪录余韵”：

1. 展示本次鱼获总数；
2. 逐条显示鱼种、尺寸和首次发现；
3. 显示当前估值，不自动出售；
4. 标出可完成订单和新纪录；
5. 显示本日剩余潜捕窗口。

### 12.4 鱼铺界面

- 当前时段和下次刷新时间；
- 各鱼种当前报价、涨跌箭头和重点需求量；
- 主要价格原因；
- 玩家鱼获、新鲜度和尺寸；
- 单条、按鱼种和全部出售；
- 批量出售的分档价格预览；
- 当前订单和截止时间；
- 最近成交与个人平均卖价。

## 13. 结算与数据接口

### 13.1 潜捕结算

```text
1. 进入时锁定区域、鱼群、尺寸和行为种子
2. 运行水下场景，只记录已成功进入鱼篓的鱼
3. 主动或强制上浮
4. 原子创建鱼获实例与海生图鉴记录
5. 消耗一个有效鱼群窗口
6. 退出活动快照并一次性提交固定1潮刻，不叠加水下现实耗时
7. 处理时段边界、鱼市与世界事件
8. 显示上岸后的最新估价
9. 创建关键活动自动存档
```

### 13.2 出售结算

```text
1. 锁定玩家选择的鱼获实例
2. 读取当前报价、需求量和批量分档
3. 生成并显示不可变出售预览
4. 玩家确认
5. 原子移除鱼获、增加金贝、增加鱼铺库存
6. 减少重点需求量并记录成交
7. 写入一个财富轨迹节点
8. 刷新鱼铺与HUD
```

任一步无法完成时不得出现“鱼已扣除但金贝未到账”或“金贝到账但鱼仍可重复出售”。

start_time: day_8_morning
duration: 12_tides
visibility: public

## 14. 存档与确定性

存档至少包含：

```yaml
daily_dive_windows_remaining: int
fish_catch_inventory: Array
marine_discoveries: Dictionary
marine_size_records: Dictionary
dive_scene_seed: int
fish_market_quotes: Dictionary
fish_market_stock: Dictionary
fish_market_demand: Dictionary
fish_market_orders: Array
fish_market_history: Array
```

约束：

- 进入潜捕场景时立即保存场景种子；
- 读档后同一区域、同一窗口生成相同鱼种、尺寸和初始路线；
- 玩家可以改变追逐对象和操作结果，但不能靠读档刷新稀有鱼；
- 已完成出售使用唯一成交编号，重复载入结算界面不得再次发钱；
- 未识别鱼种ID保留为兼容记录，不允许静默删除玩家鱼获；
- 价格历史按时段保存，不能在读档时重新计算出不同旧报价。

## 15. 分阶段范围

### P0：核心闭环

- 1个地图入口、3个潜捕区域；
- 12种鱼、3类行为；
- 50秒基础氧气、4格鱼篓、每日3个有效窗口；
- 鱼获实例、新鲜度与独立海生图鉴；
- 1个蓝鳍鱼铺收购柜台；
- 4个时段报价、需求标签、NPC供给与批量收购深度；
- 天气、时段、餐厅需求和基础订单联动；
- 同种子可复现与原子出售结算。

### P1：成长与人物

- 呼吸、脚蹼、鱼篓和保存箱升级；
- 厨师、渔民、收藏家与米娅的专属订单和记忆；
- 住宅水族箱、纪录鱼陈列和海鲜节；
- 更多海况、鱼群迁徙与外岛目击消息。

### P2：产业与长期市场

- 蓝鳍渔业经营账目与商会份契价格联动；
- 玩家供应合同、餐厅合作和大宗鱼获；
- 港口事故、船队扩张、游客周期与产业竞争；
- 玩家成为主要供应商、股东或控制人后的市场影响。

首版明确不做：水下战斗、死亡惩罚、装备耐久、鱼类繁殖模拟、几十个独立买家、逐秒价格跳动和复杂期货工具。

## 16. 自动测试契约

1. 每日有效鱼群窗口不得小于0，窗口用尽后交互不生成鱼获、不追加潜捕固定耗时；返回世界后自然时钟仍按统一规则运行。
2. 氧气不得小于0；归零后只能进入自动上浮，不能继续抓取。
3. 鱼篓容量严格限制单次捕获数量，替换鱼获不得复制实例。
4. 同一潜捕种子产生相同鱼种、尺寸和初始路线。
5. 鱼获与永久万物使用不同数据集合，出售鱼获不改变万物图鉴。
6. 新鲜度按日结稳定变化，冷藏修正不能产生永久鲜活状态。
7. 同一市场状态输入必须产生相同报价，报价始终在设计上下界内。
8. 玩家出售后鱼获数量减少、金贝增加、鱼铺库存增加，三者在同一原子结算完成。
9. 同一成交编号最多结算一次，快速重复确认不得重复发放金贝。
10. 批量出售超出重点需求量后必须使用较低边际价格，预览金额与实际金额一致。
11. 时段边界按固定顺序结算消费、NPC供给、事件和新报价。
12. 普通熟练潜捕的长期平均收益保持在设计目标区间，稀有鱼不会让保底路线稳定压过风险玩法。
13. 商店、晨报和价格解释读取同一份供需状态，不出现互相矛盾的公开事实。
14. 读档不会重掷已锁定鱼群或改写已经保存的历史报价。
15. 水下氧气按活动现实秒数下降，但世界自然时钟保持暂停；上岸只提交一次1潮刻，动画长度、暂停和帧率变化不得造成重复计时。

## 17. 原型验收

1. 新玩家无需阅读完整规则，能在首次下水中理解氧气、抓取、鱼篓和上浮。
2. 玩家能够说出至少一种“这条鱼现在为什么贵或便宜”的原因。
3. 普通玩家每日三次潜捕可以恢复低额玩法本金，但不能形成无限收入。
4. 稀有鱼更难到达或捕获，而不是只依赖极低概率弹出奖励。
5. 玩家出售鱼获后，鱼铺库存、剩余需求和后续价格产生一致变化。
6. 天气至少同时影响水下体验、NPC供给和一种公开价格解释。
7. 鱼获、海生图鉴和永久万物在界面及数据上清楚分离。
8. 完成一次“看行情—下水—抓鱼—上岸—出售”的闭环不超过5分钟。
9. 氧气耗尽不会死亡或丢失已捕获鱼，但会明确结束本次潜捕。
10. 未来接入蓝鳍渔业份契时，可以直接读取已经存在的采购、库存、销量和需求事件，不需要重做鱼市规则。

## 18. 本轮决策摘要

1. 用“海岸潜捕”整体替换浅滩采集，不保留随机点击直接发钱的旧机制。
2. 氧气限制单次探索，鱼篓限制单次携带，每日鱼群窗口限制总经济产出。
3. 氧气归零只强制上浮，不死亡、不扣钱、不丢失已入篓鱼获。
4. 鱼获是有限商品，海生图鉴是永久记录，二者都不进入72种永久万物谱系。
5. 鱼价由需求标签、真实库存、NPC供给、玩家出售、天气和事件共同形成，并按时段刷新。
6. 批量出售存在收购深度，玩家不能用顶部报价无限倾销。
7. 蓝鳍鱼铺承担鱼获交易，椰影杂货铺继续专注万物研究服务。
8. 鱼市先形成独立可玩的供需闭环，再把经营汇总接入蓝鳍渔业份契。

# 《万物之岛》背包与物品系统设计 v0.1

> 状态：首轮原型规格  
> 上位依据：总文档第4、5、6、10、15、17节  
> 关联文档：`01_万物合成系统设计_v0.1.md`、`08_商店与交易系统设计_v0.1.md`

## 1. 模块定位

背包是材料、造物、消耗品、收藏品和任务交付物的唯一事实来源。合成、商店、NPC交付和收藏都必须读取同一份实体数量，不能各自维护一套库存。

《万物之岛》不是负重管理游戏。首版背包不设置总容量和重量限制，重点是让玩家快速找到、保护、投入、使用和出售物品。

## 2. 体验目标与非目标

### 2.1 体验目标

- 玩家始终知道自己拥有多少个具体实体。
- 新材料获得后立即可用于合成、出售或交付。
- 手动锁定和软保护能避免误卖、误用和批量消耗。
- 合成投入顺序在背包到造化盆的操作中清楚可见。
- 大量材料时仍能快速筛选和批量处理。

### 2.2 非目标

- 不设置格子俄罗斯方块、负重或耐久管理。
- 不让玩家为了腾空间丢弃有价值物品。
- 不把金贝、财富头衔、关系和图谱理解做成背包物品。
- 不允许不同模块保存互相不一致的物品数量。
- 不为首版加入复杂装备栏和词条比较。

## 3. 物品类型

| 类型 | 示例 | 可堆叠 | 常见用途 |
|---|---|---:|---|
| 基础材料 | 水、土、木、盐 | 是 | 合成、出售、交付 |
| 合成造物 | 泥、陶器、静心香 | 是 | 继续合成、使用、出售、献塔 |
| 消耗品 | 饮品、辅助造物 | 是 | 牌会、竞速或探索前使用 |
| 生活物品 | 服装、小型家具 | 视类型 | 穿着、展示、消费 |
| 收藏品 | 历史牌会纪念物、纪念品 | 通常否 | 收藏展示、出售 |
| 任务/唯一物 | 关键造物、人物信物 | 否 | 故事与特殊交付 |
| 抽象材料 | 时间、信任、欲望等 | 待定 | 如果保留，必须以可计数实体存在 |

金贝显示在钱包，不占背包；祝胜券、牌桌筹码和未来份契分别由对应模块管理，但在物品历史中保留交易记录。

## 4. 物品数据结构

```yaml
item_id: mud
name: 泥
category: crafted_material
quantity: 3
stackable: true
tradable: true
usable: false
unique: false
manual_lock: false
soft_reservations:
  - recipe: pottery
newly_acquired: true
discovered_at: day_1_morning
source: synthesis
```

物品定义和玩家持有状态分开：

- 物品定义记录名称、类别、价值、用途和规则；
- 玩家状态记录数量、锁定、保护、新获得和首次来源；
- 同一物品只存在一个总数量，不按来源拆成多个不可理解的堆叠；
- 唯一物每个存档最多持有一个。

## 5. 容量与堆叠

- 首版背包没有总槽位、重量和体积上限；
- 可堆叠物品使用整数数量，不设置玩家可见的堆叠上限；
- 界面在数量较大时仍显示准确数值，不使用含糊的“999+”；
- 不可堆叠物品按独立实例保存外观、来源或故事；
- 获得物品时不会因为背包已满掉落到地面或通过邮件补发。

没有容量压力后，物品消耗主要来自合成失败、出售、使用、交付和献塔，而不是清理背包。

## 6. 物品状态

### 6.1 手动锁定

玩家可以锁定任意物品或独立实例。锁定后：

- 不能出售；
- 不能投入合成；
- 不能批量使用；
- 不能交付或献塔；
- 必须由玩家主动解除锁定。

### 6.2 软保护

系统可以为物品添加以下软保护：

- 当前追踪配方需要；
- 已接受委托需要；
- 玩家标记为收藏保留；
- 某个已装备辅助造物的最后一个实体。

软保护不会永久禁止使用。玩家选择被保护物时，需要先在背包中解除对应保护；商店和一键制作不会自动绕过保护。

### 6.3 新获得

- 首次获得的物品显示“新”标记；
- 打开详情或完成一次相关操作后清除；
- “新”只影响界面，不参与规则判断。

## 7. 获得物品

所有来源使用统一事务：

```text
验证物品与数量
→ 检查唯一物限制
→ 增加背包数量或创建实例
→ 更新首次发现与图谱
→ 更新任务、商店上架和界面标记
→ 写入物品历史
```

常见来源：

- 探索、钓鱼和拾取；
- 合成结果；
- 商店购买；
- NPC交付与委托奖励；
- 赛事或牌会特殊奖励；
- 故事事件。

如果事务中任一步失败，整次发放回滚，不能扣了金贝却没有获得物品。

## 8. 消耗物品

统一消耗顺序：

1. 验证数量是否足够；
2. 排除手动锁定与软保护数量；
3. 锁定本次需要的实体；
4. 执行合成、出售、使用、交付或献塔；
5. 成功后扣除数量并写入历史；
6. 失败时按对应模块决定回滚还是保留消耗。

不同用途的规则：

| 用途 | 失败时处理 |
|---|---|
| 合成 | 提交后全部投入实体消耗，按合成规则处理 |
| 商店出售 | 交易失败则全部回滚 |
| 消耗品使用 | 效果未成功创建则不扣除 |
| NPC交付 | 对话未完成则不扣除 |
| 献塔 | 塔纹未更新则不扣除 |

## 9. 合成操作接口

### 9.1 手动试验

- 玩家从背包按顺序选择材料，材料依次进入造化盆队列；
- 每种材料每次投入 1 个实体；
- 队列清楚显示1、2、3……的投入顺序；
- 提交前可以移除或调整顺序，不消耗实体；
- 提交后所有投入实体立即消耗；
- 结果由材料和投入顺序确定。

### 9.2 已知配方

- 图谱一键填入只选择准确配方材料；
- 自动填入不会选择手动锁定或软保护实体；
- 材料不足时列出缺少数量和已发现商店来源；
- 批量制作提供1、5、最大三个数量；
- 最大数量按每种可用材料的最小可制作次数计算；
- 批量制作不消耗潮刻。

## 10. 背包界面

### 10.1 分类

- 全部；
- 材料；
- 合成造物；
- 消耗品；
- 生活与收藏；
- 任务与唯一物。

### 10.2 排序与筛选

- 最近获得；
- 名称；
- 数量；
- 基础价值；
- 是否可合成；
- 是否可出售；
- 锁定或保护状态。

### 10.3 物品详情

显示：

- 名称、插画、数量和类别；
- 基础价值与是否可交易；
- 已知用途：合成、使用、交付、收藏、献塔；
- 首次获得时间、地点和来源；
- 锁定与软保护原因；
- 已经发现的相关配方；
- 获取来源。

不显示未发现配方的完整答案、隐藏价格变化或未获得来源的故事秘密。

## 11. 快捷操作

根据物品类型提供：

- 投入造化盆；
- 查看相关图谱；
- 使用；
- 锁定/解锁；
- 标记收藏保留；
- 追踪获取来源；
- 追踪已知配方；
- 丢弃。

首版默认隐藏“丢弃”普通可交易物品，优先引导玩家出售。任务、唯一物和已锁定物不能丢弃。

## 12. 与其他模块的接口

### 12.1 商店

- 商店读写同一背包数量；
- 批量出售排除锁定和软保护物品；
- 买回恢复原物品状态，但不自动恢复玩家已经手动解除的保护；
- 商店购买的已发现材料可立即参与合成。

### 12.2 NPC与社交

- 委托需要的物品添加软保护；
- 交付完成后原子扣除物品并创建人物记忆；
- NPC不能持有一份与玩家背包无关的“幽灵任务物品”。

### 12.3 地图与时间

- 地图拾取点通过统一事务发放物品；
- 打开和整理背包不消耗潮刻；
- 日终不会自动删除普通物品；
- 限时物品若会过期，详情页必须显示准确结算时间。

### 12.4 金贝、牌会与竞速

- 金贝和锁定本金不作为普通背包堆叠；
- 辅助造物在进入牌会或封盘前锁定，结算后按效果定义消耗或解除；
- 祝胜券由竞速模块保存，但可以从物品历史查看购买与结算记录。

## 13. 保存与物品历史

存档必须保存：

- 所有物品定义版本；
- 堆叠数量与独立实例；
- 手动锁定和软保护；
- 首次获得信息；
- 最近交易、合成、交付和使用记录；
- 正在进行的原子事务状态。

首版至少保留最近100条物品变动记录，便于调试“物品为什么消失”以及金贝和物品守恒问题。

## 14. 首版内容范围

- 无容量限制的统一背包；
- 6类物品；
- 堆叠与独立实例；
- 手动锁定、任务需要和收藏保留；
- 合成队列与投入顺序；
- 已知配方批量制作；
- 商店批量购买、出售和买回接口；
- NPC交付与辅助造物接口；
- 排序、筛选、详情和最近100条物品历史。

首版不做：重量、格子整理、耐久、装备词条、仓库扩建、跨角色共享、邮件附件。

## 15. 原型验收

1. 所有模块读取同一物品数量，不出现商店与合成显示不同库存。
2. 手动锁定物不能被出售、合成、交付或批量使用。
3. 软保护物不会被一键操作误用，玩家可以主动解除后使用。
4. 合成队列准确保存投入顺序，并按顺序得到确定结果。
5. 批量制作和批量交易在任意数量下保持物品与金贝守恒。
6. 发放或扣除失败时事务完整回滚，不产生负数数量。
7. 玩家获得100种以上物品后，仍能在10秒内通过分类或筛选找到目标。

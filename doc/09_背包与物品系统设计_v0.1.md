# 《万物之岛》万物图鉴与账户凭证系统设计 v0.1

> 文档状态：原型实施基线
> 核心原则：万物永久掌握；有限次数只用于明确的账户凭证

## 1. 模块定位

万物图鉴是全游戏对“玩家已经认识和能够使用什么”的唯一事实来源。造化盆、竞速辅助、NPC 委托、万象塔和商店都读取同一份发现状态。

首版不设置万物数量、重量、容量、堆叠、锁定或出售。玩家通过快捷键打开万物图鉴，而不是管理材料背包。

海岸潜捕新增的鱼获属于独立生活商品，使用鱼获箱与海生图鉴管理，不把普通材料背包规则重新引入永久万物系统。

## 2. 数据类型

### 2.1 永久万物

四阶原型的永久万物固定为：

- 一阶根源：水、火、土；
- 二阶反应：蒸汽、泥、熔岩；
- 三阶环境与工艺：云、沼泽、黑曜石、动力、陶器、喷发、间歇泉、黏土、山岳、泥泉、火山云；
- 四阶世界成形：55种天象、地貌、生境、材料、工艺、动力与生活万物，完整清单以`01_万物合成系统设计_v0.1.md`为唯一关系基线。

四阶原型总计72种永久万物。所有非根万物均有唯一二元来源。

永久万物只有发现与未发现状态，不记录持有数量。

### 2.2 账户凭证

账户凭证是明确允许有限次数的非合成资源，例如：

- 免费逐风体验券。
- 实验折扣剩余次数。
- 活动入场资格。
- 剧情许可与一次性邀请。

凭证具有用途、次数和失效条件，但不能放入造化盆。

### 2.3 状态与进度

以下内容属于状态，不包装成物品：

- NPC 关系与记忆。
- 委托是否开启或完成。
- 财富头衔。
- 地图区域发现。
- 万象塔楼层。
- 商店线索历史。

### 2.4 鱼获与海生记录

鱼获是唯一在本阶段明确具有数量、尺寸、新鲜度和出售价值的生活商品。每条鱼获保存物种、尺寸、捕获时间、新鲜度和来源区域，并存入独立鱼获箱。

海生图鉴是永久观察记录，保存是否见过、是否抓到、最大尺寸和已验证生态信息。海生记录不能出售、不能放入造化盆，也不计入万象塔进度。

| 数据 | 有数量 | 可出售 | 会失鲜 | 永久记录 |
|---|---:|---:|---:|---:|
| 永久万物 | 否 | 否 | 否 | 是 |
| 鱼获实例 | 是 | 是 | 是 | 否 |
| 海生图鉴 | 否 | 否 | 否 | 是 |

## 3. 永久万物状态

```yaml
discovered:
  water:
    first_source: initial
    first_day: 1
    first_tide: 1
  mud:
    first_source: synthesis
    first_day: 1
    first_tide: 3
```

原型可以用布尔字典保存是否发现，但所有业务必须通过统一接口查询：

- `is_discovered(item_id)`
- `discover_item(item_id, source)`
- `discovered_item_ids()`
- `discovered_recipe_count()`

## 4. 获得永久万物

### 4.1 初始掌握

游戏开始时只加入水、火、土。

### 4.2 世界观察

海岸潜捕可以产生鱼获、海生记录、金贝交易机会和关系线索；人物与地图事件可以提供观察、台词和消息。但四阶谱系中的新万物只能通过造化盆发现。

### 4.3 商店研究

商店只提供关系方向线索和实验折扣，不直接写入发现状态。

### 4.4 合成发现

正确二元组合生成新的永久万物，并记录发现双方、时间与首次来源。

## 5. 使用永久万物

### 5.1 万物合成

- 左右两个选择栏读取完整已发现集合。
- 使用不改变发现状态。
- 同一个万物可以同时作为左右输入。

### 5.2 逐风竞速

- 带有竞速辅助定义的万物出现在赛前辅助列表。
- 每场最多选择一个。
- 支付部署费后获得信息，万物永久保留。

### 5.3 NPC 委托

委托检查玩家是否已经理解目标万物。交付的是知识、方法、建议或现场协助，不扣除万物。

示例：阿葵要求玩家发现水罐的补水方法。玩家发现水罐后即可完成委托，水罐仍保留在图鉴中。

### 5.4 万象塔

塔层读取发现总数和关键组合，不要求献出或消耗万物。

## 6. 万物图鉴界面

图鉴、鱼获箱与物品详情属于阅读和整理界面，打开时进入`ui_paused`；浏览、筛选、比较尺寸和整理快捷入口不追加固定潮刻，关闭后不补算停留时间。

### 6.1 总览

- 已发现数量与一至四阶进度。
- 按阶层与根源、天象、地貌、生境、工艺等类别筛选。
- 最近发现。
- 拥有竞速用途或 NPC 用途的标记。
- 未发现条目只显示轮廓，不显示答案。

### 6.2 万物详情

每个已发现万物显示：

- 插图、名称、类别和层级。
- 世界描述。
- 首次发现来源和时间。
- 已验证关系数量。
- 是否仍存在未尝试关系。
- 已开放的竞速、NPC、万象塔用途。

### 6.3 快捷入口

- `I` 或 `Tab`：打开万物图鉴。
- 图鉴详情：将该万物送入造化盆左侧或右侧。
- 竞速用途：跳转逐风竞速入口说明。
- 最近实验：跳转造化盆历史。

## 7. 账户凭证界面

凭证不与万物混排。图鉴底部或账户页单独显示：

- 凭证名称。
- 当前次数。
- 使用位置。
- 是否永久、当日刷新或一次性。

首版至少显示免费逐风体验券和实验折扣次数。

账户凭证界面沿用图鉴的`ui_paused`规则，查看次数与用途不会改变日程状态。

## 8. 与其他模块的接口

| 模块 | 读取 | 写入 |
|---|---|---|
| 造化盆 | 已发现万物、尝试记录 | 新发现、尝试记录 |
| 商店 | 已发现状态、尝试记录、折扣次数 | 线索、折扣次数 |
| 逐风竞速 | 已发现竞速辅助、免费券 | 免费券次数、辅助使用记录 |
| NPC | 目标万物是否发现 | 委托状态、人物记忆 |
| 万象塔 | 发现数量与关键万物 | 塔层状态 |
| 财富系统 | 凭证消费对应的金贝变化 | 财富轨迹 |
| 海岸潜捕 | 潜捕装备、海生记录 | 鱼获实例、尺寸纪录、剩余窗口 |
| 蓝鳍鱼铺 | 鱼获实例、新鲜度、报价 | 移除已售鱼获、成交记录、鱼铺库存 |

所有模块不得自行维护第二份万物数量或发现副本。

## 9. 存档与兼容

至少保存：

```yaml
discovered: Dictionary
attempted_pairs: Dictionary
recent_synthesis_pairs: Array
free_race_ticket: int
synthesis_discount_uses: int
aqiu_request_active: bool
aqiu_request_done: bool
fish_catch_inventory: Array
marine_discoveries: Dictionary
marine_size_records: Dictionary
```

约束：

- 未识别的万物 ID 保留但不显示，避免升级时丢失。
- 读取时自动去除重复最近记录。
- 凭证次数不得小于 0。
- 发现状态只能从未发现变为已发现。
- 业务结算必须先验证后写入，避免部分成功。
- 鱼获使用唯一实例ID；已经出售的实例不得通过界面重建或重复结算。
- 未识别鱼种ID保留原始记录并禁止出售，直到兼容映射或数据修复完成。

## 10. 自动测试契约

1. 新发现永久存在并能被所有模块立即读取。
2. 合成、竞速和委托不会移除万物。
3. 图鉴不存在数量、锁定、出售和容量逻辑。
4. 同一万物重复发现不会重复发放首次奖励。
5. 左右造化栏展示同一集合。
6. 凭证次数只在对应业务成功时变化，且永不为负。
7. NPC 委托完成后目标万物仍然可用。
8. UI 清楚区分永久万物与有限凭证。
9. 开局永久万物严格为水、火、土，商店与海岸潜捕不能提前解锁谱系万物。
10. 鱼获、海生图鉴和永久万物使用不同集合，任一出售行为不得改变`discovered`。
11. 同一鱼获实例最多出售一次，新鲜度和尺寸不同的鱼不得错误合并结算。

## 11. 原型验收

- 玩家通过万物图鉴管理永久发现，通过凭证页管理有限次数服务。
- 图鉴是永久万物唯一入口和唯一事实来源。
- `I`/`Tab` 能快速查看所有已发现万物及用途。
- 造化盆、商店、竞速和 NPC 对同一发现状态即时一致。
- 免费券和实验折扣以账户凭证形式单独展示。
- 新增万物时只需扩展统一数据与素材映射，不需要为每个模块建立库存副本。
- 玩家能在界面上清楚区分永久万物、账户凭证、鱼获箱和海生图鉴四类信息。

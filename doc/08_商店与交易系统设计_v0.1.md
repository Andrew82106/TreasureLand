# 《万物之岛》商店与交易系统设计 v0.1

> 状态：首轮原型细化稿  
> 上位依据：总文档第4、5、10、11、15、17节  
> 关联文档：`01_万物合成系统设计_v0.1.md`、`03_金贝与基础经济设计_v0.1.md`、`07_地图与移动系统设计_v0.1.md`、`09_背包与物品系统设计_v0.1.md`

## 1. 模块定位

商店负责把金贝、材料与玩家亲手得到的造物连接起来。玩家可以购买原材料继续合成，也可以把背包里真实存在的采集物或造物卖给商店建立本金。服装、家具和生活消费暂不进入本版商店。

首版商店必须保证：

1. 基础合成不会因为缺货永久中断；
2. 商店只出售基础原材料，不因玩家发现造物而自动出售造物；
3. 出售和购买使用统一背包与金贝结算；
4. 不存在买入—合成—卖回的无限无风险套利；
5. 商店在NPC不在场时仍保留必要服务。

## 2. 体验目标与非目标

### 2.1 体验目标

- 玩家第一次到椰影街就能理解购买、出售、库存和价格。
- 每一种材料都有清晰插图，购买前能看懂自己买的是什么。
- 合成发现不会把结果“倒灌”进购买目录；造物的来源仍然是玩家亲手合成。
- 交易后立即看见金贝和财富头衔变化。
- 商品缺货时，玩家知道其他获取方式。

### 2.2 非目标

- 不制作讨价还价小游戏。
- 不设置必须重复跑多家商店的琐碎采购。
- 不用隐藏手续费或随机改变结算价格。
- 不让关系直接把价格降低到可以无限套利。
- 不把祝胜券窗口和牌桌买入伪装成普通商品货架。

## 3. 商店类型

| 商店/窗口 | 地点 | 购买 | 出售 | 首版状态 |
|---|---|---|---|---|
| 万物杂货铺 | 椰影街 | 仅基础原材料 | 背包里真实存在的普通材料与造物 | 完整实现 |
| 服装与生活商店 | 椰影街 | 暂不开放 | 暂不开放 | 后续版本 |
| 祝胜券窗口 | 逐风海岸 | 各类祝胜券 | 封盘前撤销 | 调用逐风竞速规则 |
| 茶摊入席处 | 椰影茶摊 | 使用金贝入席与追加桌上金贝 | 牌局结束直接结算金贝 | 调用命运牌会规则 |
| 未来高级商店 | 金潮坡、云顶区 | 艺术品、高级服装、资产 | 分类回收 | 首版仅保留入口 |

祝胜券窗口和茶摊入席处使用统一金贝结算，但不参与普通商品库存和回收价公式。

## 4. 商品数据结构

```yaml
offer_id: general_store_water
shop_id: general_store
item_id: water
base_price: 8
stock_mode: unlimited
stock: null
buy_enabled: true
sell_enabled: true
required_discovery: water
required_wealth_title: drifter
demand_multiplier: 1.0
refresh_rule: daily
```

商品报价至少包含：

- 商品与商店编号；
- 当前买入价和回收价；
- 当前库存及刷新时间；
- 发现要求；
- 财富头衔要求；
- 当日供需系数；
- 是否允许批量交易；
- 其他获取来源提示。

## 5. 商品解锁

### 5.1 基础材料

- 水、土、火、木、风、石、鱼、果实等首版基础材料从开局起可见；
- 常用基础材料使用不限量库存，避免基础合成被日程随机阻断；
- 商店购买是便利途径，探索、钓鱼、拾取和委托仍能以更低成本获得材料。

### 5.2 造物边界

- 泥、炭、雾、陶器等配方产物不进入商店采购目录；
- 玩家首次发现或大量制作一种造物，都不会让商店凭空生成同类库存；
- 造物只有在玩家背包中真实存在时，才会出现在“出售背包实物”区域；
- 出售时原子扣除背包实体并增加金贝，数量不足时交易必须失败；
- 玩家卖出的造物在本原型中不转成可购买商品，避免绕过合成与收藏过程；
- 剧情唯一物、人物专属物和不可交易概念始终不会进入交易列表。

### 5.3 财富头衔商品（后续）

以下分层不在当前原型开放；未来开放生活与资产商店时，财富头衔只控制商品购买资格，不控制进入商店：

| 头衔 | 商品范围示例 |
|---|---|
| 漂流者 | 基础材料、低价生活品 |
| 学徒 | 常用已发现材料、基础服装 |
| 小赢家 | 不常见材料、特色服装和家具 |
| 富商 | 稀有材料、收藏展示品 |
| 大亨/岛之名流 | 高级艺术品、未来资产入口 |

账户财富值跌破门槛后，尚未购买的高阶商品立即变为不可购买；已经买入背包的商品不会被收回。

## 6. 库存与刷新

| 商品类型 | 库存规则 | 刷新时间 |
|---|---|---|
| 基础必要材料 | 不限量 | 价格日更，数量不刷新 |
| 玩家背包中的造物 | 只允许出售，不形成采购库存 | 实时读取背包 |
| 生活与装饰品 | 暂不开放 | 后续版本 |
| 玩家刚出售的普通物品 | 不进入采购货架 | 立即结算 |
| 特殊限时商品 | 独立数量 | 活动开始时 |

商店库存由时间系统在日终生成并写入存档。同一日终存档反复读取不会刷新出不同商品。

## 7. 价格与回收

沿用基础经济公式：

```text
商店售价 = 基础价值 × 稀有系数 × 当日供需系数 + 便利溢价
普通回收价 = 基础价值 × 0.6 × 当日需求系数
```

规则：

- 最终价格四舍五入为整数金贝；
- 当日供需系数首版限制在0.8—1.5；
- 商店售价在玩家打开界面时锁定，关闭界面前不会突然变化；
- 普通回收价不会高于同店售价；
- 批量交易每件使用相同单价，不因数量临时加价或压价；
- 委托和限时需求使用独立报价，并明确显示购买方；
- 关系可以解锁特殊货源，但首版不提供价格折扣。

## 8. 购买流程

```text
选择商品
→ 选择数量：1、5 或最大可买
→ 显示单价、总价、剩余现金和建议恢复金
→ 确认购买
→ 原子扣除金贝并增加背包实体
→ 重新计算账户财富值、财富头衔和剩余购买资格
```

规则：

- 购买本身不消耗潮刻；
- 只有剩余现金低于建议恢复金时显示一行非阻断提醒；
- 不设置风险等级和二次确认；
- 如果付款成功但物品写入失败，整笔交易回滚；
- 购买造成头衔下降时，当前商品仍完成交付，之后不再允许购买已经关闭的高阶商品。

## 9. 出售流程

```text
从背包选择可交易物品
→ 选择数量
→ 显示单价、总回收额和保护状态
→ 确认出售
→ 原子移除实体并增加金贝
→ 更新库存、账户财富值和财富头衔
```

以下物品默认不能出售：

- 剧情唯一物；
- 当前手动锁定物；
- 不可交易概念；
- 已经进入牌桌、祝胜券或其他进行中活动的金贝。

委托需要、收藏保留和当前追踪配方使用的物品采用软保护：可以出售，但必须先在背包中解除保护。

### 9.1 误售保护

- 本原型先依靠背包锁定与委托保留阻止误售；
- 被保护物品的出售按钮禁用，并明确显示保护状态；
- “本次售出买回”留待交易事务历史完成后实现；
- 买回功能完成前，玩家卖出的造物也不会转入普通采购库存。

## 10. 商店界面

当前原型默认提供：

- 顶部店主欢迎区，直接说明“只卖材料，造物由玩家合成”；
- 三列插图货架卡，显示材料插图、名称、单价和购买按钮；
- 当前持有金贝；
- 独立的“出售背包实物”区域，只从统一背包实时生成；
- 每个出售项显示插图、持有数量、单件回收价和保护状态；
- 交易后立即刷新金贝、背包和货架按钮状态。

不显示：

- 未发现配方的完整材料关系；
- 隐藏供需事件；
- 商品未来价格预测；
- 风险等级或“推荐购买”答案。

## 11. NPC与营业状态

- 万物杂货铺清晨、白天和傍晚由店主提供服务；
- 夜晚由代班店员保留基础购买和出售；
- 店主不在场时，人物专属对话和特殊货源暂停，但必要材料不会无法购买；
- 商店界面显示当前接待者；
- 玩家与店主关系不会影响公共基础服务。

## 12. 与其他模块的接口

### 12.1 万物合成

- 商店只为合成提供基础原材料，不出售任何配方产物；
- 图谱一键制作可以显示缺失材料及其商店来源，但不会自动购买；
- 批量制作需要的材料数量来自背包实时库存。

### 12.2 背包与物品

- 所有交易通过统一物品事务完成；
- 手动锁定物不能出售；
- 商店显示背包持有量，不能维护独立副本；
- 购买和出售完成后立即更新相关任务、配方和收藏状态。

### 12.3 金贝、头衔与时间

- 每次交易后立即更新现金、账户财富值和财富头衔；
- 交易不消耗潮刻；
- 每日库存和供需由日终结算生成；
- 建议恢复金是唯一资金提醒。

### 12.4 地图与NPC

- 商店入口是地图兴趣点；
- 店主离开时由代班服务接替；
- 玩家可以从地图追踪基础材料是否由杂货铺供应，但不能远程购买。

## 13. 首版内容范围

- 万物杂货铺完整购买与出售；
- 祝胜券窗口和茶摊入席接口；
- 13种带正式插图的基础原材料；
- 背包中实际材料与造物的出售；
- 物品保护；
- 每日库存和价格刷新；
- 店主与代班服务。

首版不做：服装、家具、生活消费、造物采购、误售买回、讨价还价、拍卖、玩家摆摊、商会批发、远程购物、跨岛商品、复杂会员折扣。

## 14. 原型验收

1. 玩家第一次进入商店后能从插图理解材料，并独立完成购买与出售。
2. 基础材料不会因随机库存耗尽而永久阻断合成。
3. 合成品与衣服在任何刷新后都不会出现在采购货架。
4. 任意批量交易都满足金贝与物品实体守恒。
5. 不存在购买材料—合成—普通回收的无限无风险套利。
6. 交易导致头衔变化时，商品资格立即正确更新，已经完成的交易不回滚。
7. 店主离开时，基础商店服务仍可使用。
8. 背包中没有对应实体时，出售操作必须失败；已有造物可以正常出售。

# 《万物之岛》商店研究服务系统设计 v0.2

> 文档状态：四阶合成原型实施基线
> 首版地点：椰影杂货铺
> 核心功能：出售研究方向与实验效率，不出售谱系万物
> 关联交易模块：`14_海岸潜捕与鱼市供需系统设计_v0.1.md`

## 1. 模块定位

椰影杂货铺帮助卡住的玩家继续研究。商店不出售万物、不回收图鉴成果、不维护材料库存，也不能绕过造化盆直接完成任何阶层。

本文件中的“商店”专指椰影杂货铺研究服务。鱼获出售由独立的蓝鳍鱼铺收购柜台承担，不与研究服务共用商品目录、库存或结算逻辑。

玩家购买的只有：

- 当场交付的配方方向线索；
- 有明确次数上限的实验折扣；
- 后续与地图、竞速和收藏相关的研究服务。

## 2. 体验目标

- 玩家花钱后得到清楚、立即可用的研究价值。
- 线索缩小推理范围，但不公开完整答案。
- 商店永远不破坏“所有万物由二元关系发现”的核心成就感。
- 服务没有隐性库存、随机缺货、出售分页或重复的“买1”按钮。
- 金贝只有在服务能够完整交付时扣除。
- 商店界面以少量插图卡片呈现，不使用长商品清单。

## 3. 首版服务

### 3.1 配方方向线索

价格：12金贝。

系统从“结果未发现、组合尚未尝试、至少一个输入已发现”的关系中选择一条，交付：

- 一个已知万物名称；
- 关系语法，例如相变、聚集、塑形或容纳；
- 另一项输入的阶层与类别；
- 不公开另一项输入名称和结果名称。

示例：

```text
阿拓的线索：陶器还有一条容纳关系，对方是一阶根源类万物。
```

同一线索不连续重复。没有可交付的新线索时服务禁用且不收费。

### 3.2 三次实验折扣

价格：24金贝。

- 购买后，接下来三次尚未尝试的组合费用减半并向上取整。
- 历史组合复查不消耗次数。
- 金贝不足、未选择完整或实验没有执行时不消耗次数。
- 折扣最多储存6次；账户必须至少空出3次容量才能购买。
- 页面与造化盆始终显示剩余次数。

## 4. 商店数据

```yaml
id: recipe_hint
name: 配方方向线索
type: hint
price: 12
art: cloud
description: 公开一个已知万物、目标阶层和关系语法，不揭晓答案
repeatable: true
```

```yaml
id: experiment_discount
name: 三次实验折扣
type: discount
price: 24
art: energy
uses: 3
cap: 6
repeatable: true
```

## 5. 购买结算

1. 读取服务当前状态。
2. 预生成可交付线索，或检查折扣容量。
3. 检查当前金贝。
4. 一次性扣除金贝。
5. 立即交付线索文本或折扣次数。
6. 记录财富轨迹。
7. 刷新商店、造化盆和HUD。

任一步无法完成时不得扣款。禁止出现“先扣钱，随后发现没有线索”的部分结算。

## 6. 商店界面

### 6.1 顶部

- 阿拓与店铺主视觉；
- 当前金贝；
- 已发现万物数量；
- 剩余实验折扣次数；
- 固定说明：“这里帮助你研究关系，不出售万物。”

### 6.2 研究服务区

只展示两张大卡片：

1. 配方方向线索：名称、价格、交付范围、当前是否有新线索；
2. 三次实验折扣：名称、价格、剩余次数、容量状态。

最近购买的线索单独显示一次。商店不显示库存、数量、服装、原料和出售入口。

## 7. NPC与营业

- 阿拓在场时提供对白、研究观点和关系反馈。
- 阿拓离开时由店员继续交付两项基础服务。
- 打开商店后进入`ui_paused`；查看、比较和购买不追加固定潮刻，玩家阅读速度不影响世界时间。
- 商店不会因为日结随机关闭。
- 后续限时服务必须明确结束时间，不使用隐藏刷新。

## 8. 与其他模块的接口

### 8.1 万物合成

- 线索只读取固定配方表、尝试记录和永久发现集合。
- 线索不会写入`discovered`，也不会代替一次实验。
- 折扣由合成结算消耗，商店只增加凭证次数。

### 8.2 金贝经济

- 每次成功消费形成一个财富轨迹节点。
- 线索与折扣不折算回金贝。
- 不存在库存资产、买卖价差或回收套利。

### 8.3 NPC与世界

- NPC委托可奖励折扣次数或免费线索。
- 新区域可以增加地图研读、赛事报告或收藏陈列服务。
- 财富头衔可开放更专业的服务，但不能直接出售谱系答案。

### 8.4 蓝鳍鱼铺边界

- 椰影杂货铺不收购鱼获、不显示鱼价，也不维护供需库存；
- 蓝鳍鱼铺读取玩家鱼获、当前报价、重点需求量、订单与新鲜度；
- 阿拓可以提供保存设备研究或海况知识说明，但不能在研究页面代为收购鱼获；
- 两个店铺都允许NPC离开后由代班人员保留公共服务；
- 蓝鳍鱼铺出售结算必须同时移除鱼获、增加金贝并更新鱼铺库存；
- 研究服务消费和鱼获出售分别写入财富轨迹，不能混成一个无法解释的交易节点。

## 9. 自动测试契约

1. 商店目录只包含线索与折扣，不包含万物解锁、数量、出售或每日库存。
2. 服务购买前后永久发现数量不变。
3. 金贝不足、没有新线索或折扣容量不足时不扣款。
4. 线索交付成功后才扣款，且不选择已经完成或已验证无效的组合。
5. 线索至少包含已知万物、关系语法、目标阶层与类别。
6. 折扣严格增加三次，最多储存六次。
7. 历史组合复查不消耗折扣。
8. 商店与造化盆读取同一折扣次数和最近线索。
9. 研究商店目录不得出现鱼获、鱼价、出售按钮或鱼市库存。
10. 蓝鳍鱼铺与研究商店使用不同业务标识和结算入口，不能把鱼获误写入永久发现集合。
11. 商店打开期间世界时间停止，关闭后不补算停留时间；购买快慢不改变任何商品、NPC或市场状态。

## 10. 原型验收

- 商店只展示两项清楚、可解释的研究服务。
- 玩家能理解线索缩小了什么范围，仍需要自己完成配对。
- 所有购买即时交付并进入财富轨迹。
- 服务不会跳过任何阶层或直接增加图鉴万物。
- UI有插图、状态、用途和明确禁用原因。
- 玩家能够清楚区分“向阿拓购买研究服务”和“向蓝鳍鱼铺出售鱼获”两类相反资金流向。

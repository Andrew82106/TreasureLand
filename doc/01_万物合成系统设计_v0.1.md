# 《万物之岛》万物合成系统设计 v0.2

> 文档状态：四阶原型实施基线
> 核心规模：3 个一阶万物、3 个二阶万物、6 个三阶万物、9 个四阶万物
> 关联模块：万物图鉴、金贝经济、知识商店、逐风竞速、NPC 委托、万象塔

## 1. 模块定位

万物合成是《万物之岛》的核心发现玩法。玩家从水、火、土三个根万物开始，在造化盆中选择任意两个已经发现的万物，判断它们之间是否存在稳定关系。正确关系会永久生成新的万物；新万物立即进入左右两侧的选择库，并成为下一阶关系的输入。

系统的主要乐趣来自三件事：

1. 根据自然变化、地形演化、生活工艺和象征关系推理配方。
2. 每次发现都会扩大可探索组合空间，并在岛屿、图鉴和其他玩法中留下变化。
3. 从三个根万物逐步长出天气、生命、工艺与地貌，形成一张可理解的世界谱系。

四阶原型验证完整的发现节奏，不追求无限数量。所有关系必须先满足可解释性，再考虑数量扩张。

## 2. 核心规则

### 2.1 永久万物

- 万物一经发现便永久进入图鉴。
- 合成、委托、竞速研读和献塔不会减少万物。
- 左右两个槽位读取同一份永久图鉴。
- 同一个万物可以同时放入左右槽，但四阶原型没有同物配方。

### 2.2 固定二元关系

- 每次实验严格选择两个万物。
- 左右顺序不影响结果，`水 + 火` 与 `火 + 水` 是同一关系。
- 一个无序组合最多对应一个结果。
- 配方不读取天气、时段、位置、人物关系或随机数。
- 正确关系必定成功；无配方关系必定留下失败记录。

### 2.3 试验费

首次尝试一个组合时，根据两项输入中的最高阶收取金贝：

| 最高输入阶层 | 首次试验费 |
|---:|---:|
| 一阶 | 2 金贝 |
| 二阶 | 5 金贝 |
| 三阶 | 12 金贝 |
| 四阶 | 30 金贝 |

- 成功与失败使用同一费用口径。
- 已尝试组合永久免费复查。
- 购买实验折扣后，下一次新组合费用减半并向上取整。
- 金贝不足时不扣款、不消耗折扣、不建立尝试记录。

## 3. 四阶成长结构

```text
一阶：世界根源       3 个
  水　火　土
        ↓ 三组根关系全部成立
二阶：直接反应       3 个
  蒸汽　泥　熔岩
        ↓ 每个二阶万物产生两条三阶关系
三阶：环境与工艺     6 个
  云　动力　沼泽　陶器　黑曜石　山岳
        ↓ 六个三阶万物全部参与四阶关系
四阶：世界成形       9 个
  雨　雷暴　生命　沃土　水罐　窑炉　石刃　矿脉　河流
```

阶层代表发现深度，不代表物理强弱或商品稀有度。高阶万物可以是宏大的自然现象，也可以是建立在多层理解上的生活器物。

## 4. 关系设计语法

每条配方必须属于以下至少一种关系语法：

| 语法 | 含义 | 示例 |
|---|---|---|
| 混合 | 两者直接形成新的稳定介质 | 水 + 土 = 泥 |
| 相变 | 温度或状态改变形成新形态 | 水 + 火 = 蒸汽 |
| 冷却 | 高温物质快速失热形成新结构 | 熔岩 + 水 = 黑曜石 |
| 聚集 | 小尺度现象汇集成大尺度现象 | 蒸汽 + 水 = 云 |
| 塑形 | 材料经作用获得可使用结构 | 泥 + 火 = 陶器 |
| 地貌 | 物质与地形共同形成环境 | 山岳 + 水 = 河流 |
| 催化 | 一项提供环境，另一项提供变化动力 | 沼泽 + 动力 = 生命 |
| 容纳 | 器物与内容物共同定义用途 | 陶器 + 水 = 水罐 |
| 象征 | 自然规律经过岛民理解后形成可复用概念 | 云 + 动力 = 雷暴 |

禁止仅靠谐音、字面拼接或完全不可见的剧情条件建立主线配方。允许少量奇想关系，但结果插图、描述和失败反馈必须帮助玩家理解这条关系。

## 7. 路径覆盖审计

“80%—100% 的万物继续参与下一阶”指输入覆盖率，不指任意两个万物都有结果。若绝大多数乱配都成功，玩家不需要观察和推理，合成会退化成点击清单。

| 过渡 | 可作为该阶段核心输入的万物 | 至少有一条后续关系 | 覆盖率 |
|---|---:|---:|---:|
| 一阶 → 二阶 | 3 | 3 | 100% |
| 二阶 → 三阶 | 3 | 3 | 100% |
| 三阶 → 四阶 | 6 | 6 | 100% |

附加约束：

- 二阶每个万物恰好有 2 条三阶关系，避免某一支明显成为死路。
- 三阶每个万物至少有 1 条四阶关系，平均 2.17 条。
- 四阶的九个结果分别为未来天气、生命、工艺、地质和地图系统保留延伸口。
- 新增五阶内容时，四阶输入覆盖率不得低于 80%。

## 8. 发现、失败与提示

### 8.1 首次成功

成功结算顺序：

1. 扣除本次试验费。
2. 中央造化盆吸入左右万物。
3. 播放关系特效与新造物揭示。
4. 写入永久图鉴和组合历史。
5. 显示“新万物打开了多少条尚未尝试关系”。
6. 更新阶层进度、万象塔和相关NPC话题。

### 8.2 无效关系

失败不公布正确答案，但必须说明关系为何没有稳定：

| 失败类型 | 判定 | 示例反馈 |
|---|---|---|
| 同源静默 | 左右为同一万物 | “两者只是彼此映照，没有产生新的变化方向。” |
| 作用不足 | 两项都有潜力，但彼此没有作用机制 | “它们并存了片刻，却没有一方能够改变另一方。” |
| 尺度不合 | 一个是宏观环境，一个是缺乏作用点的根万物 | “这段关系缺少让变化集中发生的结构。” |
| 已经记录 | 组合曾经尝试 | 直接显示历史结果与解释，费用为 0 |

### 8.3 线索层级

线索从轻到重分为三档：

1. 免费方向：指出某个万物仍有几条未发现关系，以及目标大致类别。
2. 商店线索：公开其中一侧万物与另一侧的阶层或关系语法。
3. NPC事件：在对话或场景中演示自然现象，不直接展示配方按钮。

线索不会直接把未发现万物放入图鉴，也不会跳过谱系层级。

## 9. 独立页面设计

造化盆使用与命运牌会同级的全屏玩法页面，不嵌入普通弹窗。

### 9.1 固定舞台

```text
┌─────────────────────────────────────────────────────────────┐
│ 万物合成 · 造化盆   当前层进度   图鉴 12/21   金贝   组合谱 返回 │
├──────────────┬──────────────────────────┬──────────────┤
│ 左侧万物图鉴   │                          │ 右侧万物图鉴   │
│ 阶层筛选       │      大型造化盆舞台       │ 阶层筛选       │
│ 2列插图卡片    │   [左万物] 关系 [右万物]  │ 2列插图卡片    │
│ 独立滚动       │      费用 / 结果 / 解释    │ 独立滚动       │
│                │      开始实验 / 清空       │                │
├──────────────┴──────────────────────────┴──────────────┤
│ 最近发现与谱系推进：云 → 雨 / 雷暴；山岳 → 4 条待研究关系       │
└─────────────────────────────────────────────────────────────┘
```

布局约束：

- 全屏舞台基线为 `1280×720`，页面本身不整体滚动。
- 左右图鉴宽度一致，读取同一发现集合，各自拥有独立滚动区。
- 中央舞台保持固定，不因图鉴数量增加而上下移动。
- 卡片显示插图、名称、阶层和未完成关系数量；不显示物品数量。
- 已选卡片使用金色外框，另一个栏中仍可选择同一万物。
- 结果、费用、失败原因只在中央舞台出现一次。

### 9.2 阶层筛选

左右图鉴均提供：

- 全部；
- 一阶；
- 二阶；
- 三阶；
- 四阶。

尚未发现的万物只在“组合谱”里显示为对应阶层的暗格，不在可选择图鉴中显示名称和插图。

### 9.3 实验演出

| 阶段 | 时长 | 表现 |
|---|---:|---|
| 选中 | 0.12 秒 | 卡片抬起、边框点亮、中央槽同步出现缩略图 |
| 确认 | 0.18 秒 | 按钮锁定，左右卡片向盆心移动 |
| 反应 | 0.45 秒 | 水纹扩散、盆心增亮，根据结果类别改变粒子色 |
| 成功揭示 | 0.55 秒 | 新插图从盆心放大，名称、阶层和关系解释依次出现 |
| 失败反馈 | 0.35 秒 | 光线收束后熄灭，展示失败类型与下一步方向 |

动画期间屏蔽重复提交；关闭页面会立即完成数据结算，不允许靠中断动画逃避费用或重复奖励。

## 10. 组合谱

组合谱是全屏页面内的规则查询层，按一至四阶纵向展示：

- 已发现万物：完整插图、名称、描述和来源关系。
- 未发现暗格：只显示阶层、类别轮廓和该阶剩余数量。
- 已验证配方：显示两项输入、结果和关系解释。
- 已验证失败：只计入尝试统计，不在主谱系连线上占节点。

每阶标题显示 `已发现 / 总数`。完成一整阶时播放一次阶层点亮演出，并为万象塔提供对应进度。

## 11. 跨模块接口

### 11.1 逐风竞速

四阶原型提供四种情报用途：

| 万物 | 竞速研读 | 提供信息 | 是否改变赛果 |
|---|---|---|---:|
| 雨 | 雨势推演 | 当前天气对各逐风兽的修正排名 | 否 |
| 雷暴 | 雷暴听兆 | 本场随机波动等级与高波动逐风兽 | 否 |
| 水罐 | 补水观察 | 所选逐风兽巡航、耐力与稳定性比较 | 否 |
| 河流 | 河势比照 | 地形段基础表现与场内排名 | 否 |

竞速研读只读取已有赛事数据并收取服务费。同一种子、同一选择在使用与不使用研读时必须产生相同赛果。

### 11.2 NPC委托

- 阿葵首个合成委托要求发现“水罐”并分享补水方法。
- 分享方法不会移除水罐。
- 铃婆婆根据当前最低完成阶层提供下一条免费方向提示。
- NPC不能直接改变固定配方结果。

### 11.3 知识商店

商店提供配方方向线索和三次实验折扣，不直接出售谱系万物。全部 21 个万物必须通过造化盆关系发现。

### 11.4 万象塔

万象塔按阶层完成度点亮：

| 塔层 | 条件 |
|---|---|
| 潮火基座 | 发现全部 3 个二阶万物 |
| 云泥回廊 | 发现全部 6 个三阶万物 |
| 众生工坊 | 发现至少 6 个四阶万物 |
| 四象观台 | 发现全部 9 个四阶万物 |

## 13. 自动化测试契约

必须长期守住以下条件：

1. 开局图鉴只有水、火、土。
2. 三组不同根万物配对全部生成二阶结果。

## 14. 四阶原型验收

1. 玩家第一次进入页面能立即理解“左选一个、右选一个、中央实验”。
2. 前三次不同根万物组合都能形成成功发现，建立正反馈。
3. 玩家在不查答案的情况下，可以根据关系解释推导出至少一半三阶配方。
4. 每个二阶和三阶万物都有明确后续用途，不出现发现即闲置的支线。
5. 完成四阶原型时，玩家能在组合谱中清楚回看 18 条世界生成关系。
6. `1280×720` 下核心舞台、实验按钮、费用、结果和返回按钮同时可见。
7. 完整自动化测试与 GitHub Actions 通过后才允许上传主分支。

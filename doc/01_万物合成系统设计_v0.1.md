# 《万物之岛》万物合成系统设计 v0.1

> 文档状态：原型实施基线
> 关联模块：金贝经济、万物图鉴、商店服务、逐风竞速、NPC 委托、万象塔
> 核心定义：玩家以两个永久解锁的万物进行实验，支付金贝发现新的永久万物

## 1. 模块定位

万物合成是游戏的长期发现主线。玩家从岛上的自然观察、人物知识和商店研究中获得初始万物，再从左右两个图鉴栏各选择一个万物，让它们在造化盆中相遇。

一个万物被发现后即成为玩家永久掌握的知识与造物能力，可以无限次参与组合，也可以在其他系统中作为永久工具使用。

系统承担四项职责：

1. 提供清楚、快速、可持续扩展的二元组合玩法。
2. 把金贝转化为发现机会和收藏进度。
3. 让每个新发现开启新的组合关系、收藏进度和跨模块用途。
4. 把造物能力接入竞速、NPC、万象塔与后续世界事件。

## 2. 体验目标

- 玩家始终知道自己正在选择哪两个万物、实验需要多少金贝。
- 玩家把注意力放在未知组合、规律推理和图鉴扩展上。
- 成功时有完整的发现演出、插图、描述、图鉴进度和用途提示。
- 失败时保留万物，只记录组合、扣除实验费，并给出可理解的方向反馈。
- 同一个无效组合不会重复收费。
- 新发现立即出现在左右两个选择栏中，使组合空间肉眼可见地增长。
- 复杂度集中在“已知万物之间的关系”、试验选择和规律推理。

## 3. 万物数据

每个万物至少具有以下字段：

```yaml
id: stable_string_id
name: 玩家可见名称
category: 自然 | 生灵 | 工艺 | 生活 | 灵感
tier: 0 | 1 | 2 | 3
description: 图鉴描述
source: 初始 | 世界发现 | 商店知识 | 合成发现 | 剧情
art_key: 图鉴与选择栏使用的素材键
race_aid: null | 逐风竞速辅助定义
```

### 3.1 层级

| 层级 | 含义 | 典型内容 |
|---|---|---|
| 0 | 世界中可以直接认识的基础万物 | 水、土、火、风、木、石、鱼、盐 |
| 1 | 一次基础关系形成的稳定造物 | 泥、炭、雾 |
| 2 | 具有清楚用途的工艺或生活造物 | 陶器、工具、风帆、咸鱼 |
| 3 | 能显著连接其他玩法的精细造物 | 静心香、观风铃及后续高阶造物 |

层级用于组织图鉴、控制实验费用和安排发现节奏，不代表物品可以交易或堆叠。

## 4. 永久发现规则

### 4.1 初始万物

首版开局永久掌握：

- 水
- 土
- 火
- 风
- 木
- 石

鱼、盐、花和果实通过浅滩观察永久发现；金、布和纸通过商店的一次性知识服务永久发现。

### 4.2 发现状态

每个万物只有两种核心状态：

- 未发现：不显示真实名称与插图，不能参与组合。
- 已发现：永久进入图鉴，左右选择栏均可无限使用。

发现状态不下降，不因实验、委托、竞速或剧情使用而失去。

### 4.3 发现来源

- 初始认知：进入游戏即可使用。
- 世界观察：第一次在地图活动中遇见时加入图鉴。
- 人物传授：NPC 对话或委托永久开放。
- 商店知识：支付一次性费用获得某种基础知识。
- 合成发现：完成正确二元组合。

## 5. 二元合成规则

### 5.1 输入

- 每次必须选择两个已发现万物。
- 左右选择栏读取同一份图鉴。
- 同一个万物可以同时放入左右两侧，以支持 `A + A` 配方。
- `A + B` 与 `B + A` 是同一个组合。
- 输入区固定为左、右两个槽位；每个无序组合只对应一个结果记录。
- 每个无序万物对最多对应一个稳定结果。

组合键统一写为排序后的两个 ID：

```text
pair_key = min(left_id, right_id) + "|" + max(left_id, right_id)
```

### 5.2 实验费用

费用在点击实验前显示，按照两个输入中较高的层级确定：

| 输入最高层级 | 标准实验费 |
|---:|---:|
| 0 | 2 金贝 |
| 1 | 5 金贝 |
| 2 | 12 金贝 |
| 3 | 30 金贝 |

商店的实验折扣服务可令接下来三次新组合实验费用减半，向上取整。折扣只在真正产生新尝试时消耗。

### 5.3 收费时机

一次新组合按以下顺序结算：

1. 确认左右两侧均为已发现万物。
2. 生成无序组合键。
3. 检查该组合是否已经尝试。
4. 已尝试：直接展示历史结果，不扣金贝、不消耗折扣次数。
5. 未尝试：检查金贝是否足够。
6. 扣除实验费，并记录财富轨迹。
7. 判定配方、记录结果、播放反馈。

金贝不足时不建立尝试记录，玩家筹集金贝后仍可进行第一次有效实验。

## 6. 结果与反馈

### 6.1 首次成功

首次发现必须展示：

- 合成双方及关系动画。
- 新万物完整插图和名称。
- 图鉴类别与层级。
- 一段世界描述。
- 图鉴发现进度。
- 该万物是否拥有竞速、人物或万象塔用途。
- 因此新增的“可尝试组合数量”，但不直接公布答案。

结果永久加入图鉴，并立即同步到左右选择栏。

### 6.2 已知成功组合

再次选择已经成功的组合时：

- 不扣金贝。
- 显示已发现结果与首次发现记录。
- 可以回看发现演出。
- 不重复增加图鉴、财富或任务奖励。

### 6.3 新的无效组合

无效组合必须展示：

- 本次双方名称。
- 已支付的实验费。
- “无稳定结果”的明确结论。
- 一条根据类别关系生成的方向提示。
- 该组合已经永久记录，不会再次收费。

失败提示只帮助玩家理解方向，不直接公开一个正确配方。

### 6.4 已知无效组合

再次选择已经验证无效的组合时，界面直接显示历史记录和原提示，费用为 0。

## 7. 尝试记录

系统永久记录所有已完成实验：

```yaml
pair_key: water|wood
left_id: water
right_id: wood
result_id: ""
success: false
cost_paid: 2
day: 1
tide: 3
feedback_key: quiet_pair
```

记录用于：

- 防止重复收费。
- 在选择栏标记已尝试关系。
- 生成最近实验历史。
- 统计玩家的探索方式。
- 为商店线索排除已经验证过的组合。

## 8. 造化盆界面

### 8.1 固定布局

```text
┌──────────────┬────────────────────┬──────────────┐
│ 左侧万物图鉴 │  当前组合与造化盆   │ 右侧万物图鉴 │
│ 搜索/分类     │  左万物 + 右万物    │ 搜索/分类     │
│ 已发现卡片    │  费用/折扣/实验按钮 │ 已发现卡片    │
└──────────────┴────────────────────┴──────────────┘
│ 最近尝试、无效记录、发现进度与当前提示                         │
└───────────────────────────────────────────────────────────────┘
```

- 左右栏内容、排序和卡片规格完全一致。
- 点击左栏卡片只替换左侧选择，点击右栏卡片只替换右侧选择。
- 中央始终显示双方插图、名称、组合是否尝试过、当前费用和玩家金贝。
- 实验按钮只有在双方均选择且金贝足够时可用；历史组合费用为 0。
- 选择完成后不自动执行，防止误扣费。

### 8.2 卡片状态

- 普通：已发现、尚未在当前侧选中。
- 选中：清楚高亮并显示“左侧”或“右侧”。
- 有新关系：该万物至少存在一个双方均已发现但未尝试的正确关系，只显示微弱提示。
- 已穷尽：在当前已发现集合内已没有未尝试关系。

### 8.3 最近实验

显示最近八次不同组合：

- 两个万物。
- 成功结果或“无稳定结果”。
- 首次支付费用。
- 尝试发生的时间。

## 9. 首版配方

所有配方均为无序二元组合：

| 左/右万物 | 另一万物 | 结果 | 层级 | 首版用途 |
|---|---|---|---:|---|
| 水 | 土 | 泥 | 1 | 继续合成陶器 |
| 火 | 木 | 炭 | 1 | 继续合成静心香 |
| 风 | 水 | 雾 | 1 | 图鉴与后续天气研究 |
| 泥 | 火 | 陶器 | 2 | 万象塔生活陈列 |
| 木 | 金 | 工具 | 2 | 竞速分段量具 |
| 布 | 风 | 风帆 | 2 | 竞速天气读势 |
| 鱼 | 盐 | 咸鱼 | 2 | 阿葵委托知识 |
| 炭 | 花 | 静心香 | 3 | 竞速稳定性观察 |
| 金 | 风 | 观风铃 | 3 | 竞速天气优势观察 |

## 10. 与其他模块的接口

### 10.1 金贝经济

- 新组合实验是稳定金贝消耗口。
- 已知组合和回看不收费。
- 首次发现不直接发放金贝，价值体现在图鉴、用途和塔进度。
- 实验费必须纳入财富轨迹和经济日志。

### 10.2 商店

- 商店可永久解锁特定基础万物。
- 商店可提供配方方向线索和限次实验折扣。
- 商店不出售万物份数，也不回收合成成果。

### 10.3 逐风竞速

- 部分造物拥有信息型竞速辅助。
- 造物本身永久保留，每场最多部署一个。
- 部署费以金贝支付；辅助只改变玩家可见信息，不暗改赛果。

### 10.4 NPC 与万象塔

- 委托检查“是否已经发现目标万物”，不扣除目标。
- NPC 可以为发现提供基础万物、线索或实验费报销。
- 万象塔根据永久发现数量与关键造物组合点亮楼层。

## 11. 数据与存档

至少保存：

```yaml
discovered:
  water: true
  earth: true
attempted_pairs:
  water|earth:
    success: true
    result_id: mud
    cost_paid: 2
synthesis_discount_uses: 0
recent_synthesis_pairs: [water|earth]
```

必须保证：

- 同一组合不会因左右位置交换被记录两次。
- 读档不会重复发放首次发现奖励。
- 图鉴与左右选择栏读取同一份发现状态。
- 配方表升级后，已有成功记录仍指向稳定 ID。

## 12. 原型验收

1. 左右两侧展示完全相同的已发现万物集合。
2. 玩家只能选择两个万物，且允许选择两个相同万物。
3. 合成不减少任何万物，也不存在数量不足。
4. 新组合只扣金贝；费用在执行前可见。
5. `A + B` 与 `B + A` 共用记录和结果。
6. 正确组合永久解锁结果，结果立即可参与下一次实验。
7. 无效组合有原因、提示和永久记录。
8. 已尝试组合再次执行不扣金贝。
9. 金贝不足时不扣款、不建立记录。
10. 至少四种造物能够在逐风竞速中提供不同信息。
11. 商店中的万物商品都是永久知识，服务商品使用明确的凭证次数。
12. 自动测试覆盖永久性、费用、无序组合、重复免单、商店服务和竞速辅助。

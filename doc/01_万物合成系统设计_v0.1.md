# 《万物之岛》万物合成系统设计 v0.3

> 文档状态：四阶完整内容基线
> 核心规模：3个一阶万物、3个二阶万物、11个三阶万物、55个四阶万物
> 固定关系：69条无序二元配方
> 关联模块：万物图鉴、金贝经济、研究商店、逐风竞速、NPC委托、万象塔

## 1. 模块定位

万物合成是《万物之岛》的核心发现玩法。玩家从水、火、土三个根万物开始，在造化盆中选择任意两个已经发现的万物，判断它们是否能形成稳定关系。成功关系生成一个独立的新万物，新万物永久进入图鉴，并立即加入左右两个选择库。

系统不维护材料数量。合成不会消耗输入万物，只收取首次关系实验的金贝费用。关系结果固定、左右顺序无关，不读取天气、时段、人物或随机数。

## 2. 四阶规模公式

### 2.1 覆盖率定义

本系统中的“层间覆盖率”定义为：

```text
本阶新增成功关系数 ÷ 进入本阶前累计万物的无序两两组合数
```

令：

```text
N_t = 第t阶新增万物数
S_t = 完成第t阶后的累计万物数
p_t = 从前一阶进入第t阶的关系覆盖率

N_t = ceil(p_t × C(S_(t-1), 2))
S_t = S_(t-1) + N_t
```

覆盖率是最低目标，因此非整数时向上取整。已经使用过的输入对不能产生第二个结果；新增配方必须从尚未占用的输入对中选择。

### 2.2 四阶计算

| 阶层 | 覆盖率 | 计算 | 新增万物 | 累计万物 | 累计关系 |
|---|---:|---:|---:|---:|---:|
| 一阶 | 初始 | 固定 | 3 | 3 | 0 |
| 二阶 | 100% | `ceil(1.00 × C(3,2))` | 3 | 6 | 3 |
| 三阶 | 70% | `ceil(0.70 × C(6,2))` | 11 | 17 | 14 |
| 四阶 | 40% | `ceil(0.40 × C(17,2))` | 55 | 72 | 69 |

最终规模固定为：

```text
3 → 3 → 11 → 55
72个万物，69条成功关系
```

## 3. 核心规则

### 3.1 永久万物

- 万物一经发现便永久保留。
- 左右两侧读取同一份图鉴，同一万物可以同时放入两侧。
- 委托、竞速研读、献塔和商店服务都不能移除万物。
- 四阶内容中没有同物配方，但同物实验允许执行并留下失败记录。
- 四阶万物是当前版本终点。任一侧放入四阶万物时，中央明确提示五阶尚未开放，不能提交、不扣款、不消耗折扣、不写入失败记录。

### 3.2 固定关系

- 每次实验严格选择两个万物。
- `水 + 火`与`火 + 水`视为同一无序输入对。
- 一个无序输入对至多对应一个结果。
- 每个非根万物恰好拥有一个首次来源。
- 正确关系必定成功；无配方关系必定失败。
- 同一输入对成功或失败后永久免费复查。

### 3.3 首次实验费用

| 最高输入阶层 | 首次实验费 |
|---:|---:|
| 一阶 | 2金贝 |
| 二阶 | 5金贝 |
| 三阶 | 12金贝 |

金贝不足时不扣款、不消耗折扣、不写入记录。实验折扣只对真正执行的未尝试组合生效，费用减半并向上取整。

只走69条成功关系、不产生失败时，完成当前四阶谱系的基础实验费为`3×2 + 11×5 + 55×12 = 721金贝`。

## 8. 关系设计语言

配方必须至少符合一种可观察关系：

- 物理变化：混合、相变、冷却、烧结、磨蚀、压实；
- 地貌过程：侵蚀、堆积、冲击、抬升、滞水、导流；
- 气象过程：聚集、对流、悬浮、放电、降水、卷扬；
- 生态过程：滋养、孕育、适应、富集、固岸；
- 工艺过程：塑形、模制、容纳、围火、蓄压、做功。

禁止仅靠谐音、字面拼接或不可见剧情条件建立主配方。允许岛屿幻想化表达，但结果名称、插图和解释必须让玩家能回推关系。

## 9. 成功、失败与提示

### 9.1 首次成功

1. 原子扣除实验费；
2. 左右万物进入中央造化盆；
3. 播放关系类型对应的反应演出；
4. 揭示新插图、名称、阶层和因果；
5. 写入永久图鉴与组合历史；
6. 更新组合谱、万象塔、NPC话题和跨模块用途。

### 9.2 失败

失败不公开正确答案，但必须提供：

- 失败类型；
- 为什么两项万物没有稳定改变彼此；
- 建议保留哪一侧、改换什么类别或阶层；
- 已支付费用与永久免费复查状态。

### 9.3 线索

1. 免费方向：某万物当前还有多少条可直接研究关系；
2. 商店线索：公开一侧万物、关系语法以及另一侧阶层和类别；
3. NPC演示：通过场景现象表达关系，不直接写出答案。

商店和NPC不能直接把未发现结果写入图鉴。

## 11. 组合谱

组合谱按阶层展示：

- 每阶`已发现/总数`；
- 已发现万物的名称、类别、描述与插图；
- 已验证关系的两项输入、结果、关系类型和因果；
- 未发现节点只显示暗格，不泄露名称；
- 已验证失败进入历史，不占成功谱系节点。

四阶55项内容应支持按类别折叠，避免一次显示过长清单。后续五阶只能在当前72项全部完成数据审计后增加。

## 13. 数据契约

万物定义至少包含：

```yaml
id: steam_engine
name: 汽动机
tier: 4
category: 机械
description: 蒸汽被动力结构约束并转化为持续机械运动。
art_source: creation
art_index: 4
```

配方定义至少包含：

```yaml
inputs: [steam, energy]
output: steam_engine
relation: 做功
logic: 蒸汽被动力结构约束并转化为持续机械运动。
```

运行状态至少保存：

```yaml
discovered: Dictionary
attempted_pairs: Dictionary
recent_synthesis_pairs: Array
synthesis_discount_uses: int
last_shop_hint_key: String
```

## 14. 自动测试契约

1. 开局图鉴严格只有水、火、土。
2. 万物规模严格为`3、3、11、55`，总数72。
3. 配方规模严格为`3、11、55`，总数69。
4. 覆盖率按向上取整公式分别为100%、70%、40%。
5. 69个输入对唯一，69个输出唯一。
6. 每条配方输入阶层低于结果阶层。
7. 按配方表顺序可以从三个根万物完整发现全部72项。
8. 左右交换命中同一历史且不重复收费。
9. 成功、失败、折扣和金贝不足遵守原子结算。
10. 四阶作为当前终点，涉及四阶输入的选择不能提交且不会产生经济或历史变化。
11. 商店线索不能直接解锁万物。
12. NPC委托与竞速研读不消耗万物。
13. 全屏页面在`1280×720`下不溢出，中央舞台不滚动。
14. 组合谱必须显示72项、69条关系和四阶进度。

## 15. 四阶验收

- 玩家开局连续三次不同配对全部成功；
- 完成二阶后，新的12种组合中只有1种无效；
- 进入四阶后，新组合实际成功率约45%，失败能够形成下一步方向；
- 72个万物都有名称、类别、描述、插图映射和唯一来源；
- 69条关系都有关系类型与内在因果；
- 全部内容能够通过组合谱回看；
- 本地阻断测试与GitHub Actions通过后才允许进入主分支。

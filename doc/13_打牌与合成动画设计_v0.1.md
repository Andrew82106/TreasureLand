# 《万物之岛》打牌与合成动画设计 v0.1

> 文档状态：可实施动画设计基线  
> 设计日期：2026-07-15  
> 适用范围：命运牌会、万物合成、相关音效、特效、表现队列与动画测试  
> 上位规则：`01_万物合成系统设计_v0.1.md`、`02_命运牌会系统设计_v0.1.md`、`11_美术风格语言与资产规范_v0.1.md`  
> 规则边界：本文件只设计表现，不新增合成条件、不改变牌序、下注、金贝、配方、随机数或结算规则。

## 1. 设计结论

两套核心玩法需要的不是更多无差别闪光，而是更清楚、更有性格的“动作因果”：

- 命运牌会的情绪曲线是：**窥兆 → 试探 → 施压 → 解契**。玩家应看见牌、人物与金贝如何在桌上真实移动，并从动作节奏感到人物正在判断。
- 万物合成的情绪曲线是：**置物 → 相遇 → 关系发生 → 新物显形**。玩家应看懂两项万物如何形成结果，而不是只看到界面闪一下。
- 所有关键演出统一使用“**预备—发生—余韵**”三拍结构。预备告诉玩家焦点在哪里，发生表达规则动作，余韵让结果被读懂。
- 动画的主要价值是表达状态变化；装饰粒子和屏幕特效只能放在第三优先级，不能遮住卡面、人物、金额与结果。

### 1.1 一句话体验目标

| 场景 | 玩家应该产生的感觉 | 禁止滑向 |
|---|---|---|
| 命运牌会 | “这群岛民正在月夜茶亭里借水火天象订立一场有性格的财富契约” | 霓虹赌场、筹码喷泉、通用卡牌手游 |
| 万物合成 | “我亲手让两项已理解的万物在造化盆中发生关系，并发现世界的新解释” | 抽卡开箱、随机炼金、纯UI弹窗 |

## 2. 参考仓库中采用的方法

本方案研究了 `thedivergentai/GD-Agentic-Skills` 的以下内容：

- [`godot-genre-card-game`](https://github.com/thedivergentai/GD-Agentic-Skills/tree/main/skills/godot-genre-card-game)：牌不应在区域间瞬移，应沿可读路径移动；回合状态、牌面数据和视觉表现分离。
- [`godot-tweening`](https://github.com/thedivergentai/GD-Agentic-Skills/tree/main/skills/godot-tweening)：并行动画与顺序动画组合、曲线路径、旧 Tween 中断、节点绑定、统一参数资源。
- [`godot-animation-player`](https://github.com/thedivergentai/GD-Agentic-Skills/tree/main/skills/godot-animation-player)：固定时间线使用轨道、离散事件帧与音效同步，动画结束后必须能恢复默认状态。
- [`godot-particles`](https://github.com/thedivergentai/GD-Agentic-Skills/tree/main/skills/godot-particles)：一次性粒子要安全回收，运行时重播使用 `restart()`，不能依赖任意 Timer 猜测释放时机。
- [`godot-audio-systems`](https://github.com/thedivergentai/GD-Agentic-Skills/tree/main/skills/godot-audio-systems)：音效分总线、同类声音限制并发、短音效池化，避免同帧堆叠失真。

不直接照搬的内容：

- 不采用通用卡牌游戏的全息卡、彩虹稀有度或赌场筹码语言。
- 不用静态人物肖像的缩放、旋转、上下浮动冒充人物动画。
- 不为每次普通点击加入屏幕震动、全屏闪白或高强度粒子。
- 参考仓库当前以 Godot 4.7+ 为基线；实现前仍以本项目实际版本和 Compatibility 渲染路径为准，不直接复制版本特有接口。

## 3. 当前实现差距

以下是2026-07-15代码快照的表现事实，用于说明改造重点，不覆盖后续代码变更：

| 系统 | 当前已有 | 主要差距 |
|---|---|---|
| 合成 | 提交后中央区域约0.24秒变色；重建后中央面板缩放回弹 | 输入没有入盆过程；成功与失败几乎同一种反馈；关系类型没有视觉语法；结果缺少显形与首次发现层次 |
| 牌会 | 新天象牌横向缩放翻入；行动条淡入并滑动6px；NPC逐席思考 | 发牌、投入、退契、解契和派彩缺乏实体路径；卡牌只“出现”不“来自牌匣”；五名NPC中只有老乔进入真实逐帧牌桌动画切片 |
| 共同 | 已有动画锁、测试开关与延迟重建基础 | 缺少统一表现事件队列、速度档、降低动态模式、音效事件帧和特效回收策略 |

现有 `synthesis_table.gd` 的延迟重建与 Tween 清理是必须保留的安全基线。新动画不能在按钮 `pressed` 信号仍在发射时释放当前UI树。

## 4. 共用动画语言

### 4.1 三拍结构

| 拍点 | 时长建议 | 作用 | 示例 |
|---|---:|---|---|
| 预备 | 0.08—0.24秒 | 收拢注意力，展示动作来源 | 当前席位压暗周围、两项万物轻微提起 |
| 发生 | 0.18—0.70秒 | 展示规则动作本身 | 牌从牌匣飞到席位、材料沿弧线进入造化盆 |
| 余韵 | 0.12—0.55秒 | 让数字、结果和情绪被读懂 | 金贝数字滚动、结果描边稳定、人物回到待机 |

三拍不要求每次都变长。高频动作应该压缩余韵；首次发现、稀有命象和大额派彩才允许延长。

### 4.2 运动规则

- **牌、金贝、万物卡**属于有实体感的对象，位置变化必须有来源和落点，优先使用弧线或带轻微旋转的路径。
- **面板、标签、金额**属于界面信息，只做2—6px位移、明度、描边或短促缩放，不做大幅飞入。
- **人物**使用 `AnimatedSprite2D` 的真实姿态帧；Tween只能辅助焦点、灯光和覆盖层，不能扭曲人物本体。
- 高频落点使用 `EASE_OUT + TRANS_QUAD/QUART`；仪式性汇聚使用 `EASE_IN_OUT + TRANS_CUBIC`；小幅回弹只用于卡牌落桌、造物显形和按钮确认。
- 同一属性同一时刻只允许一个 Tween。新动作接管前先终止旧 Tween，再从当前值开始。
- 所有移动对象设置中心轴、正确 `z_index` 和最终精确落点；动画结束必须把位置、缩放、旋转、透明度写回确定值。

### 4.3 焦点规则

- 同一时刻只能有一个第一视觉焦点。
- 非行动席位保留低频待机，但降低对比10%—18%，不能全部静止成截图，也不能一起跳动。
- 关键文字和数字先于装饰特效出现；玩家应先读懂“谁做了什么”，再看到余韵。
- 水、火、成功、失败、加契、退契不能只靠颜色表达，必须同时使用形状、方向或文字。

### 4.4 速度与辅助选项

必须提供三套表现配置，逻辑代码只接收同一份已结算结果：

| 配置 | 用途 | 规则 |
|---|---|---|
| 标准 | 首次体验和默认游玩 | 按本文时长执行 |
| 快速 | 重复牌局与熟练玩家 | 时长乘0.55，停顿最多0.18秒，保留动作来源与落点 |
| 降低动态 | 对大幅运动敏感的玩家 | 取消弧线飞行、回弹和连续漂浮，改为0.10—0.16秒淡入、描边与数字变化 |

开发测试还可使用“关闭演出”，但关闭演出不能改变任何玩法状态。长演出允许按住确认键加速，不允许通过多次点击重复提交行动。

标准、快速、降低动态与关闭演出只改变表现时间，不改变世界时间成本。牌会演出始终包含在牌会的`activity_snapshot`中；合成仅在新实验原子结算后固定推进0.25潮刻，结果动画与玩家停留时间不再追加潮刻。

## 5. 命运牌会动画

### 5.1 牌桌整体气氛

牌桌是月夜海边茶亭，不是竞技场。环境常驻动效采用低频、不同步循环：

- 桌边灯火每2.8—4.2秒轻微明暗一次，幅度不超过6%。
- 海面反光缓慢横移，不能扫过卡面文字。
- 檐下贝饰偶尔晃动1—2帧；NPC衣摆、胡须和呼吸由各自逐帧待机承担。
- 轮到某席时，桌面命纹从桌心向该席亮起一段，不使用全屏聚光灯。

### 5.2 一手牌局的完整演出

| 阶段 | 标准时长 | 演出 |
|---|---:|---|
| 立席 | 0.45秒 | 司契标记沿桌边移动到新位置；大小盲席位的名牌依次亮起 |
| 起始投入 | 每席0.22秒，错峰0.08秒 | 金贝从对应钱包数字附近沿短弧线落入桌心；底池数字在最后一枚落下时滚动到新值 |
| 发命牌 | 单张0.26秒，间隔0.07秒 | 从桌边“未解命匣”按真实座位顺序发两轮；牌飞行时背面朝上，落桌后轻微压纸回弹 |
| 藏命行动 | 单席0.55—0.96秒 | 当前人物观察、思考、动作、对白、金贝移动按队列执行 |
| 初兆/交汇/定命 | 单张0.42秒，间隔0.10秒 | 天象牌先背面落入公共槽，再以横向收窄—切换正面—展开完成翻牌；水火播放不同余韵 |
| 解契 | 普通1.15秒，稀有最高2.00秒 | 依次翻开仍有资格者命牌，描出其采用的四张牌与命象，最后才派彩 |
| 收桌 | 0.45秒 | 未解牌归入命匣、命纹降暗、下一手司契标记开始移动 |

发牌动画必须按六席真实顺时针次序，不得为了画面方便固定从老乔开始，也不能把玩家排除在发牌与轮换之外。

### 5.3 天象牌翻开

翻牌分为四个可读步骤：

1. 牌背从命匣沿桌面弧线进入公共槽，0.22秒。
2. 牌在原槽横向收窄到8%，0.10秒；收窄到最小时切换正面。
3. 正面展开到100%，0.16秒，最后6%使用轻微回弹。
4. 播放类别余韵并稳定卡面，0.18—0.28秒。

水牌余韵：槽底出现一圈向外扩散的圆形水纹，刻度由下向上依次亮起。  
火牌余韵：牌底出现向上收束的三角焰纹，刻度由下向上短促点亮。

势阶只影响余韵强度和音高，不改变牌的实际大小。普通天象不震屏；只有首次见到“归一”或超大底池解契时允许桌面命纹产生1—2px的局部冲击，人物和UI不跟随抖动。

### 5.4 玩家与NPC行动

每次真实行动统一经过：

```text
席位命纹亮起
    ↓
人物逐帧思考（仅NPC）
    ↓
动作姿态 + 行动条更新
    ↓
金贝 / 命牌执行实体动作
    ↓
一句性格对白
    ↓
人物回到待机，焦点交给下一席
```

| 行动 | 实体动作 | 余韵 |
|---|---|---|
| 静观 | 人物手指或视线轻点天象区，桌面产生一圈很淡的命纹 | 行动条写“静观”，不飞金贝 |
| 跟契 | 一小束金贝从席位沿弧线滑入底池 | 金贝接触桌心时发出一次贝壳轻碰，底池数字滚动 |
| 立契/加契 | 人物动作幅度大于跟契；金贝分两束先后进入桌心 | 桌心命纹向其他仍需响应者扩散，明确“压力已传递” |
| 全部投入 | 钱包剩余额度形成单束明显但不遮牌的金贝流 | 席位边框变为固定“全部投入”纹样，退出后续思考队列 |
| 退契 | 两张命牌向内合拢、压暗并滑入席位后方的未解区，不翻面 | 席位饱和度下降，行动条写“退契”，退出队列 |
| 赢得底池 | 底池命纹分流到赢家席位，金贝分批回收 | 钱包先显示旧值，再随最后一束金贝到达滚动到新值 |

金贝是本作真实货币，不另造赌场筹码。动画中的金贝只表现一笔金额的方向和规模，不要求1金贝对应1枚图标：1—4枚表示小额，5—9枚表示中额，10—16枚表示大额；精确金额始终由数字承担。

### 5.5 五名NPC的动作性格

| 人物 | 思考节奏 | 下注动作 | 退契/胜利动作 |
|---|---|---|---|
| 老乔 | 先看天象，再缓慢捻须或轻叩桌沿；停顿最长的一拍放在动作前 | 手掌平推金贝，路径短而稳 | 退契时把牌整齐压住；胜利只轻点桌面收契 |
| 阿拓 | 身体前倾快，目光先扫底池数字 | 指尖一拨，金贝分两束快速入池；加契动作最外放 | 退契时耸肩后收牌；胜利时短促拍桌但不夸张 |
| 米娅 | 先在笔记上划一笔，再对照公共天象 | 金贝移动前有一次指向对应天象的动作 | 退契后继续记录；胜利时确认命象再露出小幅笑意 |
| 榕奶奶 | 先整理袖口或茶盏，动作从容 | 金贝沿最平缓的弧线进入底池 | 退契像结束一次劝告；胜利时把金贝轻轻拢回 |
| 旅人洛沙 | 斗篷先遮住手部，目光停留时间不规则但有上限 | 金贝从斗篷边缘出现，动作最难读 | 退契时牌像被雾收走；胜利只抬眼，不做大幅庆祝 |

思考阶段不显示对白；动作发生后才说一句。若同一NPC因加契再次获得行动权，使用更短的“回应动作”，不完整重复第一次思考循环。

### 5.6 新命象与解契

玩家自己的命象变化可以即时提示，但不泄露对手：

- 被采用的两张命牌与两张天象牌依次亮起描边，四张之间出现一条连续命纹。
- 命象名称从右侧原有位置淡入并上移3px，不在桌心再弹一个重复大标题。
- 命象没有提高时不播放降级或失败动画，只安静更新说明。

解契按以下顺序：

1. 冻结新的操作输入，但规则与记录按钮仍可查看。
2. 按真实解契顺序逐席翻开仍有资格者命牌；每席0.28—0.40秒。
3. 该席采用的四张牌被同色命纹串联，命象名称显示在席位行动条；不复制到中央。
4. 所有有资格者展示后，赢家命纹保持明亮，其余命纹柔和熄灭。
5. 主池与各边池按结算数据逐层派彩；无人跟注返还从桌心外圈原路回到对应席位。

三种结算强度：

| 结算 | 演出 |
|---|---|
| 退契获胜 | 不翻赢家命牌；底池直接沿单条命纹回收，0.65—0.85秒 |
| 普通解契 | 完成逐席命象对比后派彩，1.15—1.55秒 |
| 稀有命象/大额底池 | 增加0.25—0.45秒桌面关系纹余韵和独立音色，总时长不超过2.00秒 |

“归一”表现为四张完全相同的牌纹向中心合成一个稳定圆印；“轮转”表现为水火纹沿四张牌交替环行；“既济”表现为水火两条对称曲线在中心等量闭合。其他命象只使用统一描边和名称，不为九种命象全部堆叠全屏特效。

### 5.7 牌会音效

当前仓库没有运行时音频资产，首批最小音效包如下：

| 音效组 | 数量 | 用途 |
|---|---:|---|
| 纸牌 | 4 | 抽牌、落桌、翻水牌、翻火牌 |
| 金贝 | 3 | 小额、中额、大额落桌；通过轻微音高变化避免机械重复 |
| 人物动作 | 3 | 衣料、木桌轻叩、笔记划写；按人物选择使用 |
| 命象 | 4 | 普通成象、命象提升、稀有命象、解契完成 |
| 环境 | 3条循环 | 夜海、茶亭风铃、低频人声/衣料环境层 |

牌桌音效走 `SFX/Table`，按钮走 `UI`，环境走 `Ambience`。同一帧最多播放4个金贝碰撞声，超出部分合并为一次中/大额音色。

## 6. 万物合成动画

### 6.1 场景隐喻

造化盆不是抽奖机，而是一个会回应关系的观察器。它有三个稳定状态：

| 状态 | 造化盆表现 | 玩家理解 |
|---|---|---|
| 空闲 | 盆面极慢流动，边缘纹路保持低亮度 | 可以选择万物 |
| 就绪 | 左右输入对应的两段盆缘分别亮起，盆面朝中心形成两股很弱的流向 | 两项万物已放好，可以验证关系 |
| 发生 | 盆缘收亮，所有装饰降暗，焦点只留给输入、关系和结果 | 系统正在展示这次关系 |

天气、时段、人物和地点只能改变背景气氛，绝不能改变配方是否成功，也不能作为动画中的“缺失条件”。

### 6.2 选择万物

选择动作必须把“图鉴中的永久理解”与“本次放入的输入”连接起来：

1. 被点击卡片上移2px，描边亮起，0.08秒。
2. 生成一张只用于表现的卡片代理，从原卡位置沿弧线飞向对应输入槽，0.18秒。
3. 左侧输入顺时针轻转2度落位，右侧输入逆时针轻转2度落位，然后回正，0.10秒。
4. 对应半边盆缘亮起；若两项都已选，中央关系提示从“等待两项”切换为“可以验证”。

不应为选择一次万物重建整棵UI树再用缩放掩盖跳变。实现时应尽量让固定舞台、输入槽与表现覆盖层长期存在；图鉴列表更新和最终信息更新走局部刷新。若当前版本暂时仍需重建，代理卡必须位于独立覆盖层，并在延迟重建完成后才落入新槽位。

### 6.3 提交后的标准三拍

```text
预备 0.18—0.24秒
两张输入卡轻微后撤，盆缘收亮，按钮进入已确认状态
        ↓
发生 0.45—0.85秒
两项万物沿不同路径进入盆中，按关系动势相遇
        ↓
余韵 0.25—0.55秒
成功则新物显形并写入图鉴；失败则关系散开并留下“未成”记录
```

提交后立即由逻辑层原子结算一次，并生成不可变结果快照；表现层根据快照播放成功、失败、复查或阻断动画。动画期间不再次调用合成逻辑。玩家看到的旧金贝数字可保留到扣费拍点再滚动，实际状态不得延迟结算或重复结算。

### 6.4 七种关系动态语法

69条配方不制作69段独立动画。现有61种 `relation` 标签归入7个表现家族；标签只选择视觉语法，不改变规则。

| 动态家族 | 覆盖关系标签 | 核心运动 | 视觉结果 |
|---|---|---|---|
| 汇合与约束 | 聚集、汇聚、混合、调和、蓄积、蓄压、容纳、约束、封堵、固岸、附着、嵌合、熔合、铺展、分层、裹挟 | 两股流线先绕行再向内闭合 | 圆环、包覆、层叠或容器轮廓 |
| 热能与相变 | 相变、冷凝、冷却、增温、保温、聚热、围火、烧结、烘结、蒸干、激发、催化、做功、转化、放电 | 一侧注入能量，中心亮度或形态发生阶段变化 | 蒸汽、凝珠、热浪、短弧电纹；不做全屏闪白 |
| 流动与侵蚀 | 导流、渗流、渗压、冲刷、侵蚀、磨蚀、悬浮、降水、对流 | 主流线绕过或穿过另一项，留下方向性轨迹 | 沟槽、水痕、漂浮颗粒、下降雨线 |
| 迸发与抬升 | 喷发、释放、冲击、卷扬、抬升 | 先压缩到中心，再向上或向外一次性展开 | 局部喷流、尘圈、上升轮廓；强度受阶层限制 |
| 地貌与累积 | 地貌、沉积、堆积、富集、压积、压实、配土、整平、滞水、侵入 | 颗粒或层片由下向上累积，最后压稳 | 地层、台阶、岸线、稳定底座 |
| 塑形与工艺 | 塑形、模制 | 输入先成为柔软轮廓，再被看不见的模具边界校正 | 轮廓从不规则到清晰，伴随一声木/陶轻响 |
| 生长与适应 | 生境、孕育、滋养、适应 | 中心先出现微小脉动，再沿分枝路径向外生长 | 芽、菌落、湿地纹或生命光点，避免通用爱心 |

每条配方还读取两项输入的基础媒介色与输出类别，用于选择色板和少量粒子形状。例如“水+火→蒸汽”使用热能与相变语法，水纹被火纹加热后上升为低饱和蒸汽；“水+土→泥”使用汇合与约束语法，两股质感在盆面螺旋混合后沉稳下来。

### 6.5 成功显形

成功分为首次发现和历史复查：

#### 首次发现

1. 关系运动在盆心形成一个与输出外形近似的暗色剪影，0.16秒。
2. 剪影由中心向边缘显出手绘造物，0.22—0.34秒；不能只缩放整张图片。
3. 输出卡上浮6px并落回结果座，贝壳金描边只闪一次，0.18秒。
4. “新发现 · 名称”出现在中央唯一结果区；类别、关系和因果在0.08秒后依次出现。
5. 一枚很小的图鉴暗格在页脚由暗转亮，提示永久发现；不弹出第二个遮挡页面。

阶层影响仪式长度而不是“稀有抽卡”强度：

| 输出阶层 | 总时长 | 差异 |
|---:|---:|---|
| 二阶 | 0.95—1.10秒 | 关系清楚、结果快速落位，保持开局连续正反馈 |
| 三阶 | 1.10—1.30秒 | 多0.12秒剪影辨认与类别余韵 |
| 四阶 | 1.25—1.55秒 | 关系轨迹更完整，图鉴点亮更明确，但不全屏庆祝 |

#### 历史复查

输入以更短路径进入盆缘，不完整重复发现仪式；0.45—0.65秒内显示已知结果和“已验证关系”。复查仍应有明确因果，但不能让熟练玩家每次等待1.5秒。

### 6.6 失败与阻断

失败不是爆炸或红叉，而是一次可理解的“关系没有稳定”：

1. 两项输入按照各自媒介接近盆心。
2. 中心纹路尝试闭合但出现一个清晰缺口，0.18秒。
3. 两股轨迹分别退回左右半盆，饱和度轻微下降；盆面恢复平静，0.25秒。
4. 中央显示“这两项暂未形成稳定关系”，并在历史位置盖一个小型“未成”印记。

失败总时长0.65—0.85秒，不震屏、不把整页染红、不表现为设备损坏。玩家支付的是实验成本，不应被动画嘲讽。

以下情况属于阻断，不播放入盆动画：

- 金贝不足：金贝数字与费用字段各亮一次，提交按钮向左/右各移动2px后回位，总计0.18秒。
- 任一输入为空：空槽边缘按缺失侧亮起，不闪整个中央面板。
- 涉及当前终点四阶输入：五阶未开放说明淡入，输入保持原位，不扣费、不写记录。

### 6.7 造化盆音效

首批最小音效包：

| 音效组 | 数量 | 用途 |
|---|---:|---|
| 放入 | 3 | 水质、矿土/陶质、能量/火质的入盆短音 |
| 关系 | 7 | 对应七种动态家族的核心音色 |
| 结果 | 4 | 首次成功、复查成功、未成、操作阻断 |
| 图鉴 | 2 | 新暗格点亮、阶层完成 |
| 环境 | 1条循环 | 造化盆低频水/风环境，不与牌桌环境共用 |

音效必须在动作事件帧触发：卡落槽、两项相遇、剪影开始显形、输出落位、未成印记落下。禁止从按钮点击开始播放一条固定长度音效，再要求动画迁就音频。

## 7. 表现架构

### 7.1 逻辑与表现的边界

```text
玩家输入
   ↓
逻辑层一次性校验并结算
   ↓
返回结果快照 + 有序表现事件
   ↓
场景内 PresentationQueue 逐项播放
   ↓
关键结果已可读后恢复输入
   ↓
非关键余韵可在后台完成并安全回收
```

- `game_state.gd` 负责规则、顺序、金额和结果；不能等待人物动作或粒子完成后才决定结果。
- `poker_table.gd` 与 `synthesis_table.gd` 负责读取表现事件并播放；不能在动画回调里二次计算输赢或重新调用合成。
- 关闭、快速、降低动态模式消费同一事件，只改变表现器，不改变事件内容。
- UI显示可以使用演出前快照维持悬念，但结算数据必须已经确定，且只提交一次。

### 7.2 表现事件建议

牌会事件最小类型：

```yaml
type: deal_card | reveal_omen | actor_focus | actor_action | shell_transfer | fold_cards | reveal_fate | pattern_link | settle_pot
actor_seat: 0..5
target_seat: 0..5 | table
amount: int
card_ids: Array[int]
action_text: String
pattern: String
```

合成事件最小类型：

```yaml
type: place_input | converge_inputs | play_relation | reveal_creation | mark_unstable | update_collection
left_id: String
right_id: String
output_id: String
relation_family: String
is_first_discovery: bool
cost: int
```

事件数据不保存 Node 引用。表现器在播放时根据稳定槽位ID寻找当前节点，避免延迟重建后继续持有已释放对象。

### 7.3 Godot组件选择

| 需求 | 组件 | 原因 |
|---|---|---|
| 动态起点/落点、金额路径、卡牌弧线 | Tween + `Path2D/PathFollow2D` 或曲线采样 | 位置由当前布局决定，不能预烘焙 |
| 人物逐帧姿态 | `AnimatedSprite2D` / `SpriteFrames` | 保持真实像素帧和固定脚底基准 |
| 固定仪式、事件帧、音效同步 | `AnimationPlayer` | 时间线可视化，事件帧稳定，便于美术调整 |
| 一次性水纹、火星、尘粒 | 小规模 `GPUParticles2D`；必要时Compatibility回退 | 统一一次性资源，按 `finished` 回收 |
| UI数值变化 | `tween_method` 或专用计数器 | 精确显示起止金额，不创建大量标签 |

不建议当前阶段引入全屏后处理、实时模糊、复杂湍流粒子或每个配方独立Shader。

### 7.4 生命周期硬约束

1. 所有长期引用的 Tween 在新动作、关闭场景和重建前检查有效性并 `kill()`。
2. 场景局部 Tween 使用 `bind_node()` 或由目标节点创建；目标释放后不得继续回调。
3. 每次开始一条表现队列增加 `presentation_generation`；每次 `await` 返回后先检查代数、节点有效性与场景可见性。
4. 按钮回调只提交事件并安排延迟刷新；不得在信号发射栈内 `free()` 当前按钮树。
5. 一次性粒子使用 `one_shot`、`finished` 和对象池；重播使用 `restart()`，不用任意时长 Timer 直接释放。
6. `AnimationPlayer` 为所有会改动的属性提供 `RESET` 状态；离散方法轨道只调用表现事件，不承载规则结算。
7. 动画被跳过或场景被关闭时，必须执行“应用最终视觉状态—停止音效—回收代理对象—清空输入锁”四步收尾。

## 10. 自动测试与验收

### 10.1 逻辑一致性

- [ ] 同一随机种子在标准、快速、降低动态和关闭演出下产生完全相同的牌序、NPC决策、投入、底池和赢家。
- [ ] 同一合成输入在四种表现配置下产生相同结果、扣费、历史和图鉴状态。
- [ ] 连续快速点击提交只结算一次，不重复扣金贝或重复写历史。
- [ ] 跳过解契动画后，主池、边池、返还、服务费和奇数金贝分配保持守恒。

### 10.2 队列与生命周期

- [ ] 牌会逐席演出顺序与 `npc_action_sequence` / 真实行动顺序一致。
- [ ] 已退契和已全部投入席位不会进入后续思考队列。
- [ ] 测试通过真实按钮 `pressed.emit()` 触发动作，并至少等待一帧，覆盖信号发射生命周期。
- [ ] 动画中关闭页面、打开规则、切换快速模式和跳过演出均不产生已释放对象、悬挂Tween或永久输入锁。
- [ ] 合成页面在代理卡飞行、结果显形和粒子播放期间不会同步释放信号源节点。

### 10.3 视觉可读性

- [ ] `1280×720`下所有移动路径不越出固定舞台，卡牌、人物、金额和结果不会被特效遮挡。
- [ ] 同一时刻只有一个第一视觉焦点；底池数字、人物行动条和结算摘要不互相重复抢焦点。
- [ ] 水火、成功失败和行动类型在灰度截图中仍能通过形状、方向与文字区分。
- [ ] 人物本体不使用非整数像素缩放做情绪反馈；逐帧动画脚底与席位基准不漂移。
- [ ] 降低动态模式不包含大幅飞行、回弹、屏幕抖动或无限漂浮。

### 10.4 性能预算

- [ ] P0阶段同屏一次性粒子系统不超过8个，单系统粒子建议12—36个。
- [ ] 高频金贝动画使用代理对象池，不能按精确金额创建数百节点。
- [ ] 音效使用播放器池并限制同类并发，牌桌一次动作不实例化多个临时音频节点。
- [ ] 牌桌和合成各连续运行20次核心流程，无明显帧尖峰、节点数持续增长或音效叠加失真。

## 11. 本轮决策摘要

1. 牌会的吸引力来自实体牌路、人物性格、金贝流向和解契关系，不来自赌场特效。
2. 合成的吸引力来自两项万物发生可读关系；继续坚持“正确材料直接生成对应结果”，不加入契机、天气、人物或概率门槛。
3. 69条配方按现有61种关系标签归为7个动态家族，以较小资产量形成可扩展差异。
4. 固定人物演出用逐帧动画，动态路径用Tween，固定仪式与音效事件帧用AnimationPlayer，一次性粒子统一回收。
5. 逻辑先原子结算，表现只揭示结果；动画关闭、加速、降低动态或被跳过都不能改变玩法。
6. 所有新动画必须继承现有延迟重建、Tween中断和真实信号测试边界。

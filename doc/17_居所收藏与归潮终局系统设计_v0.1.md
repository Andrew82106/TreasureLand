# 《万物之岛》居所收藏与归潮终局系统设计 v0.1

> 状态：首版运行规格
> 上位依据：总文档长期目标；`03_金贝与基础经济设计_v0.1.md`、`04_时间日程与结算设计_v0.1.md`、`06_NPC与基础社交系统设计_v0.1.md`、`09_背包与物品系统设计_v0.1.md`、`14_海岸潜捕与鱼市供需系统设计_v0.1.md`
> 核心目标：让财富、永久万物、真实鱼获、人物关系和各项玩法经历最终落回一处可以被看见的岛上生活，并形成明确但不封档的终局

## 1. 系统定位

居所是长期消费、收藏陈列和人物来访空间，不是另一种可变现金融资产。

它解决四个问题：

1. 玩家赚到金贝后拥有明确、可见且不会继续复利的生活消费；
2. 永久万物除了继续合成，还能成为个人陈列；
3. 纪录级或观赏鱼除了出售，还能成为不可变现的水族收藏；
4. 万象塔完成后出现一场读取全岛经历的终局事件，而不是只显示一个头衔。

居所不出售衣服，不维护家具背包，不把永久万物变回耗材，也不提供会制造经济雪球的固定收益。

## 2. 居所等级

| 等级 | 名称 | 升级费 | 账户财富要求 | 永久万物要求 | 万物陈列槽 | 水族槽 | 每日来访 |
|---:|---|---:|---:|---:|---:|---:|---:|
| 0 | 漂流小屋 | 0 | 0 | 3 | 1 | 0 | 0 |
| 1 | 潮木居所 | 400 | 500 | 10 | 3 | 1 | 0 |
| 2 | 风灯庭院 | 2,500 | 3,000 | 30 | 6 | 3 | 1 |
| 3 | 四象宅邸 | 12,000 | 20,000 | 55 | 10 | 6 | 1 |

升级必须同时满足：

- 已拥有前一级；
- 当前可用金贝不少于升级费；
- 升级前账户财富达到要求；
- 永久发现数达到要求。

升级费直接从当前金贝扣除，记入财富轨迹的“居所改善”分类。居所、陈列和水族收藏不计入账户财富或净资产，不可抵押、转卖或用于牌桌和份契资格。

## 3. 万物陈列

### 3.1 数据规则

- 陈列只保存永久万物ID；
- 只能陈列已经发现的万物；
- 同一万物在同一居所只出现一次；
- 陈列、撤下和更换不收费、不消耗潮刻；
- 陈列不会移除万物图鉴，也不影响造化盆、委托、赛事研读和万象塔；
- 槽位数量由居所等级决定，降级不存在。

### 3.2 界面

居所页显示：

- 已使用槽位与总槽位；
- 每项陈列的插图、名称、阶层、类别和首次发现时间；
- 可陈列的已发现万物；
- “陈列”与“撤下”动作。

首次进入只显示空槽与当前可陈列集合，不自动替玩家选择。

## 4. 水族收藏

### 4.1 可放入范围

鱼获实例满足任一条件即可进入水族箱：

- 鱼种标签包含“观赏”；
- 鱼种标签包含“收藏”；
- 尺寸为“纪录级”。

放入时从鱼获箱移除该条真实鱼获，并把完整实例转入水族收藏，包括：

- `catch_id`
- 鱼种
- 尺寸
- 捕获日与潮刻
- 捕获区域
- 捕获时天气

水族收藏不再参与：

- 鱼铺出售；
- 鱼市订单；
- 新鲜度与批量加工；
- 鱼获保守变现值；
- 公司采购和库存统计。

这是明确的收藏消费选择，不是免费保存箱。玩家可以把水族收藏放流以腾出槽位；放流不会返还金贝，也不会把鱼送回鱼获箱。确认按钮必须写明“放流后不可出售或取回”。

### 4.2 展示

每条水族收藏显示鱼种、尺寸、捕获地点、捕获日、稀有度与一条生活化描述。空水族槽保留可见轮廓，使升级价值可被理解。

## 5. 人物来访

### 5.1 条件

人物来访从风灯庭院开始开放。邀请必须满足：

- 当前居所等级至少为2；
- 人物已经认识；
- 关系至少达到“熟悉”；
- 当天尚未邀请其他人物；
- 当前不是日终停驻；
- 当前可用金贝至少30。

### 5.2 结算

邀请一次：

- 支付30金贝待客支出；
- 统一推进0.5潮刻；
- 写入一条人物记忆和居所来访历史；
- 首次来访该居所的人物获得2点关系；
- 重复来访只产生新日期记录和不同对白，不重复增加关系；
- 任何随机对白只改变表现，不改变费用、时间与关系结算。

邀请不能在牌局、潜捕或赛事活动中提交。若0.5潮刻跨到日终，世界按统一时间顺序停驻，来访仍只结算一次。

## 6. 归潮盛典

### 6.1 终局原则

“万象之主”头衔仍只由完整四阶谱系产生，不读取财富门槛。归潮盛典是头衔之后的岛上故事终局，读取多个模块已经发生的事实。

终局不会删除存档，不会清空资产，不会关闭玩法。完成后进入“归潮之后”状态，玩家可以继续合成复查、投资、牌会、竞速、潜捕、社交和完善收藏。

### 6.2 盛典条件

| 见证 | 完成条件 |
|---|---|
| 万象见证 | 四象观台点亮，72种万物与69条关系完整 |
| 安居见证 | 居所达到风灯庭院或更高 |
| 陈列见证 | 至少陈列3项永久万物 |
| 海生见证 | 水族箱至少收藏1条合格鱼获 |
| 人情见证 | 至少完成1次人物来访 |
| 牌会见证 | 至少完成1场非教学牌会 |
| 逐风见证 | 至少完成1场正式逐风赛事 |
| 生计见证 | 至少完成1笔鱼铺出售 |

所有条件读取现有正式状态，不建立终局专用副本，不允许用调试按钮或手工存档字段替代。

### 6.3 触发与结算

条件全部满足后，万象塔开放“举行归潮盛典”。

提交时：

1. 创建活动前自动存档；
2. 固定读取当前居所、陈列、水族、人物、牌会、赛事、鱼市和财富摘要；
3. 推进1潮刻；
4. 将`finale_completed`永久设为真；
5. 保存完成日、潮刻和摘要；
6. 为六名核心人物写入同一事件的个人记忆；
7. 展示终局总结与“继续留在岛上”按钮。

终局不发放大额金贝，不修改份契价格、比赛结果或牌序，不重复奖励。重复进入万象塔只能回看已经保存的盛典摘要。

## 7. 归潮之后

终局后继续显示开放目标：

- 升到四象宅邸；
- 填满万物陈列；
- 建立全部12种海生记录并完善水族收藏；
- 达到岛之名流财富头衔；
- 与更多核心人物达到信任；
- 继续完成每日赛事、牌会、经营和公开关系复查。

这些是长期目标，不再制造第二个强制结局。

## 8. 跨模块口径

| 模块 | 居所读取 | 居所写入 |
|---|---|---|
| 合成与图鉴 | 永久发现、阶层、插图、首次发现 | 陈列ID，不修改发现状态 |
| 海岸潜捕与鱼市 | 真实鱼获实例、标签、尺寸、捕获信息 | 从鱼获箱转入水族或放流 |
| NPC与社交 | 认识状态、关系、人物资料 | 来访历史、首次关系变化、人物记忆 |
| 经济 | 当前金贝、账户财富、财富轨迹 | 升级费、待客费、不可变现收藏 |
| 时间 | 当前潮刻、日终停驻 | 来访0.5潮刻、终局1潮刻 |
| 命运牌会 | 非教学场次摘要 | 终局只读 |
| 逐风竞速 | 正式赛事历史 | 终局只读 |
| 万象塔 | 塔层与完整谱系 | 终局入口与回看 |
| 存档 | 版本8字段 | 居所、陈列、水族、来访与终局摘要 |

## 9. 存档字段

版本8至少保存：

```yaml
home_level: 0
home_display_items: []
home_aquarium: []
home_guest_history: []
home_last_invite_day: 0
finale_completed: false
finale_day: 0
finale_tide: 0
finale_summary: {}
```

版本6及更早存档迁移为漂流小屋、空陈列、空水族、无来访和未完成终局。迁移不得从旧鱼获或旧发现中自动替玩家建立收藏。

## 10. 验收标准

1. 四级居所的价格、要求和槽位固定，升级只扣一次。
2. 居所消费不计入可变现净资产，也不提供固定收益。
3. 永久万物陈列不会消耗、复制或改变发现状态。
4. 水族收藏只能接收符合标签或纪录尺寸的真实鱼获。
5. 转入水族的鱼获不会同时出现在鱼获箱、出售预览或订单资格中。
6. 放流不可逆且不返还金贝。
7. 人物邀请每天最多一次，支付30金贝并推进0.5潮刻。
8. 首次来访关系奖励不能重复获得。
9. “万象之主”仍只读取完整谱系。
10. 八项盛典见证全部读取正式模块状态。
11. 盛典只完成一次，固定保存摘要并推进1潮刻。
12. 完成后可以继续游玩全部正式模块。
13. 版本8存档完整保留居所、陈列、水族、来访和终局状态。
14. 旧版存档迁移不虚构任何陈列、鱼获或终局经历。

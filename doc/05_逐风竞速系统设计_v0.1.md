# 《万物之岛》逐风竞速系统设计 v0.1

> 文档状态：原型实施基线
> 核心体验：根据公开属性、天气、造物情报和赔率判断一场八兽四段赛事
> 首版造物边界：只提供信息，不直接修改赛果

## 1. 模块定位

逐风竞速是以信息判断为核心的风险活动。玩家选择一匹逐风兽、选择祝胜券类型和投入金贝，可以在确认前部署一个已经发现的竞速造物，获得额外分析信息。

比赛结果由逐风兽属性、天气和固定随机源决定。造物辅助只改变玩家在下注前看到的信息，不改变属性、随机数、赔率或最终排名。

## 2. 体验目标

- 每场比赛有可阅读、可比较的公开信息。
- 玩家能够解释自己为什么选择某匹逐风兽。
- 合成发现为竞速提供新观察方式，而不是固定属性加成。
- 不同造物解决不同信息问题，不存在全场景最强工具。
- 赛事过程通过四段排名变化提供反馈，未命中也能复盘。
- 投入规模受到财富比例限制，不能依靠单一低赔率选项指数刷钱。

## 3. 一场赛事流程

1. 显示天气、八匹逐风兽公开属性和当前赔率。
2. 选择目标逐风兽。
3. 选择不使用造物，或从已发现辅助中部署一个。
4. 显示辅助费用与新增情报。
5. 选择祝胜券类型和投入金贝。
6. 最终确认总支出。
7. 结算起步、巡航、地形、冲刺四个阶段。
8. 显示完整排名、所选逐风兽各阶段名次、派彩与账户变化。
9. 消耗 1 潮刻并记录财富轨迹。

## 4. 逐风兽数据

```yaml
id: cloudfin
name: 云鳍
speed: 78
stamina: 62
burst: 74
stability: 70
course: 82
rider: 68
form: 5
```

| 属性 | 主要影响 |
|---|---|
| 速度 | 巡航与后程基础表现 |
| 耐力 | 巡航、地形和冲刺维持能力 |
| 爆发 | 起步与冲刺 |
| 稳定 | 随机波动与强风适应 |
| 场地 | 地形段与阵雨适应 |
| 骑手 | 四阶段的小幅综合修正 |
| 近况 | 本场整体加减值 |

公开界面显示速度、耐力、爆发和稳定。场地、骑手、精确阶段权重、波动值与天气收益可以通过造物或公开报道进一步理解。

## 5. 四段模拟

### 5.1 阶段基础

```text
起步 = 爆发×0.45 + 稳定×0.30 + 骑手×0.25
巡航 = 速度×0.50 + 耐力×0.25 + 场地×0.15 + 骑手×0.10
地形 = 场地×0.35 + 稳定×0.30 + 耐力×0.20 + 骑手×0.15
冲刺 = 爆发×0.40 + 耐力×0.35 + 速度×0.15 + 骑手×0.10
```

### 5.2 天气

- 晴：无额外修正。
- 阵雨：场地属性高于 70 的部分形成正修正，低于 70 形成负修正。
- 强风：稳定属性高于 70 的部分形成正修正，低于 70 形成负修正。

### 5.3 波动

```text
阶段标准差 = 12 - 稳定×0.08
```

每一阶段独立抽取波动并累计。测试和调试必须允许固定随机种子重放。

## 6. 信息层级

### 6.1 默认公开

- 当前天气。
- 八匹逐风兽名称。
- 速度、耐力、爆发、稳定。
- 近况的概括文字。
- 独胜与已解锁票种赔率。
- 玩家当前金贝、投入上限和赛事耗时。

### 6.2 造物辅助信息

首版提供四种永久辅助：

| 造物 | 赛场称呼 | 部署费 | 提供的信息 |
|---|---|---:|---|
| 工具 | 分段量具 | 4 金贝 | 所选逐风兽四阶段基础分及各阶段场内排名 |
| 风帆 | 风帆读势 | 5 金贝 | 所选逐风兽在当前天气下的修正与天气适应排名 |
| 静心香 | 静心观测 | 6 金贝 | 所选逐风兽的波动等级、精确标准差和稳定排名 |
| 观风铃 | 全场观风 | 8 金贝 | 当前天气下收益最高的三匹逐风兽及原因 |

### 6.3 部署规则

- 只有已经永久发现的造物可以选择。
- 每场最多部署一个。
- 造物不消耗，不从图鉴移除。
- 点击比赛前显示部署费和完整情报。
- 开始比赛时与票面投入一起检查金贝并扣款。
- 免费体验券不免除部署费。
- 辅助不修改随机种子、逐风兽分数、赔率或派彩。
- 玩家可以不使用辅助，免费查看默认公开信息。

## 7. 祝胜券与赔率

### 7.1 首版票种

- 独胜：所选逐风兽获得第一名。
- 入席：所选逐风兽进入前三；账户财富达到 500 金贝后开放。

### 7.2 赔率

赔率使用当前模拟公式下的大样本真实命中概率：

```text
参考赔率 = 目标返还率 ÷ 命中概率
目标返还率 = 0.88
```

赔率限制在 1.01—20.0。造物只解释概率来源，不改变赔率。

### 7.3 投入限制

```text
单场投入上限 = min(当前金贝, 5000, max(50, 账户财富×10%))
```

- 最低投入 10 金贝。
- 免费体验券使用固定 10 金贝票面。
- 正式投入超过上限时自动限制，并在确认和结算中明确显示。
- 若选择辅助，总可用金贝必须同时覆盖部署费和票面投入。

## 8. 结算

结算顺序：

1. 扣除造物部署费。
2. 使用免费体验券或锁定票面投入。
3. 完成四段模拟。
4. 计算名次与是否命中。
5. 释放锁定本金并发放派彩。
6. 推进 1 潮刻。
7. 形成一个可解释的财富轨迹节点。

结果页必须显示：

- 造物名称、部署费和获得的情报。
- 所选逐风兽最终名次。
- 四阶段领先者与所选名次。
- 完整排名。
- 票面投入、赔率、派彩、辅助费和本场净变化。

## 9. 与万物合成的闭环

```text
发现工具/风帆/静心香/观风铃
→ 永久开放对应竞速观察方式
→ 使用信息判断天气、阶段优势和波动
→ 选择祝胜券与投入
→ 赛后复盘验证判断
→ 获得金贝或新的研究目标
```

后续造物优先扩展新的信息角度，例如拥堵、骑手配合、历史分段和赛道变化。只有在信息玩法稳定后，才考虑能够影响比赛的干预型造物。

## 10. 自动测试与平衡

### 10.1 正确性

- 未发现造物不能出现在辅助列表。
- 部署只扣金贝，不改变发现状态。
- 同一随机种子下，使用与不使用信息造物的排名完全相同。
- 部署费、票面、派彩和最终账户变化严格对应。
- 免费体验券不会免除辅助费。
- 每场只允许一个辅助。

### 10.2 概率

- 每种天气至少模拟 100,000 场校准独胜和前三概率。
- 单个票种的长期理论返还不得超过 0.97。
- 雾步入席等显眼策略不得形成稳定指数增长。
- 改变逐风兽属性、天气公式或随机波动后必须重新生成概率表。

### 10.3 信息价值

试玩记录：

- 各辅助使用率。
- 使用辅助后玩家选择是否改变。
- 辅助费用占平均投入比例。
- 玩家能否在赛后解释输赢。
- 是否出现某个辅助在所有天气和选择中被无脑使用。

## 11. 原型验收

1. 玩家可在下注前选择一个已发现造物或不使用。
2. 四种造物提供不同、明确且可验证的信息。
3. 界面提前显示部署费、信息内容和总支付能力。
4. 造物使用后永久保留。
5. 辅助不会暗中改变赛果与赔率。
6. 结果页能够复盘四段过程与完整资金变化。
7. 自动测试验证辅助权限、费用、不消耗和赛果独立性。

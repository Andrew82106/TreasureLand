# 《万物之岛》逐风竞速系统设计 v0.1

> 文档状态：首版运行基线；独立全屏赛场、中央八兽判断、同页观赛与结算已实现，正式赛兽逐帧素材与声音仍属资产扩展
> 核心体验：在真实可见的八兽赛场中，根据周围信息形成判断，再观看判断在比赛中被验证
> 首版造物边界：只提供信息，不直接修改赛果

## 1. 模块定位

逐风竞速是以信息判断为核心的风险活动。玩家选择一匹逐风兽、选择祝胜券类型和投入金贝，可以在确认前部署一个已经发现的竞速造物，获得额外分析信息。

比赛结果由逐风兽属性、天气和固定随机源决定。造物辅助只改变玩家在下注前看到的信息，不改变属性、随机数、赔率或最终排名。

逐风竞速必须拥有独立的全屏“逐风赛场”页面。它不是把赛程、八兽属性、下拉框和结果纵向堆入通用弹窗，而是把八匹逐风兽和赛道作为中央固定舞台，把天气、近况、历史、骑师、票池和造物情报布置在舞台四周。玩家的判断对象始终是中央可见的逐风兽，信息只负责帮助玩家理解它们。

## 2. 体验目标

- 每场比赛有可阅读、可比较的公开信息。
- 玩家能够解释自己为什么选择某匹逐风兽。
- 合成发现为竞速提供新观察方式，而不是固定属性加成。
- 不同造物解决不同信息问题，不存在全场景最强工具。
- 赛事过程通过四段排名变化提供反馈，未命中也能复盘。
- 投入规模受到财富比例限制，不能依靠单一低赔率选项指数刷钱。
- 玩家进入页面后第一眼看到的是逐风兽与赛场，不是表格、下拉框或长段规则说明。
- 点击任一逐风兽，周围所有信息围绕同一选择同步更新，玩家不需要在多个页面记忆编号。
- 点击“开始赛马”后在同一页面直接进入正式比赛，赛前、比赛与赛后之间不打开新的结果弹窗。
- 玩家即使没有命中，也能判断自己是信息不足、判断失误、赔率不值，还是在合理判断下遭遇波动。

## 3. 一场赛事流程

1. 玩家从赛事大厅、祝胜券窗口或公共看台进入独立的全屏“逐风赛场”。
2. 页面中央展示本场八匹逐风兽、骑师、号码、待机动作与实际赛道；四周显示今日赛程、天气、风向、赛道、近况、历史、票池和当前预计返还。
3. 玩家直接点击中央逐风兽完成选择，选中对象留在视觉中心，周围信息同步切换。
4. 玩家可选择不使用造物，或点击一张已经发现的赛前研读卡，查看费用、来源和新增判断。
5. 玩家选择祝胜券类型与投入金贝，底部决策条持续显示目标、票种、预计返还、研读费、票面、总支出与1潮刻耗时。
6. 点击“开始赛马”后进行最终校验并创建活动前自动存档；按钮在条件不足时明确显示原因。
7. 接收封盘前NPC追加票，锁定最终票池、封盘赔率与赛事随机种子。
8. 同一中央赛场切换为比赛状态，依次表现起步、巡航、地形、冲刺四段，并持续突出玩家选择的逐风兽。
9. 冲线后在同一页面进入赛后状态，显示完整排名、各段变化、派彩、账户变化与判断复盘。
10. 完成一次1潮刻结算，记录财富轨迹、赛事历史、人物记忆与晨报摘要，再允许返回赛场日程或世界地图。

### 3.1 每日赛事

- 每天固定生成清晨、白天、傍晚、夜晚四场，鸣钟潮刻为3、7、11、15。
- 每个时段只开放对应的一场；结算后不能在同一时段反复刷新新赛事。
- 未参加的旧时段赛事标记为“已错过”，不会在进入页面时补发随机赛果。
- 每场的赛事编号、阵容、天气、风向、赛道说明、开盘票池和随机种子由当日种子确定。
- 进入新的一天后生成新的四场快照；读取同一存档不会重掷。

### 3.2 页面入口与时间状态

- 进入赛事大厅的交互点后加载独立页面，不使用通用模态框承载核心操作。
- 赛前查看、选兽、阅读情报和调整投入属于`ui_paused`，玩家思考多久都不推进自然世界时间。
- 点击“开始赛马”并通过最终校验后进入`activity_snapshot`。此后退出、加速或跳过演出都不能撤销票据或重掷赛果。
- 玩家在封盘前返回世界，不扣金贝、不消耗造物研读费、不推进固定潮刻；返回位置保持在赛事大厅入口。
- 比赛和赛后首次结算完成后固定推进1潮刻，不叠加页面停留或演出的现实时间。

### 3.3 全屏页面空间结构

`1280×720`基线布局如下：

```text
┌──────────────────────────────────────────────────────────────┐
│ 今日赛事 / 时段 / 天气 / 风向 / 赛道 / 鸣钟状态 / 返回       │
├────────────┬───────────────────────────────┬───────────────┤
│ 今日赛程   │                               │ 选中逐风兽    │
│ 四场状态   │      中央逐风赛场固定舞台     │ 肖像与标签    │
│            │  8匹逐风兽、骑师、号码、赛道 │ 近况与历史    │
│ 公开消息   │  选中描边、待机、起跑与冲线  │ 场地与天气    │
│ NPC判断    │                               │ 当前赔率      │
├────────────┴───────────────────────────────┴───────────────┤
│ 造物研读卡 / 票种 / 投入 / 总支出 / 预计返还 / 开始赛马     │
└──────────────────────────────────────────────────────────────┘
```

空间规则：

- 中央赛场占页面可用面积的50%—60%，永远不进入滚动容器，也不能被侧栏挤成一条装饰横幅。
- 顶栏只放本场环境和页面级操作；左栏负责全场公共事实，右栏只解释当前选中的逐风兽。
- 底部是唯一决策与支付区；选择、费用、预计返还和“开始赛马”不得散落在多个面板。
- 低优先级的完整公式、32场历史与详细票池进入抽屉或次级页，不能遮住赛场。
- `1600×900`与`1920×1080`增加赛道留白与观众层次，不扩大信息密度；`1280×720`下核心舞台和决策条无需滚动即可完整操作。

### 3.4 三种页面状态

| 状态 | 中央舞台 | 周边信息 | 主要操作 |
|---|---|---|---|
| 赛前判断 | 八兽在检录区或起跑线待机，允许直接点击 | 全部可读，随选中对象更新 | 选兽、研读、票种、投入、开始赛马 |
| 正式比赛 | 切换为实际赛道与奔跑动画，镜头跟随赛群 | 左右栏降暗或收起，只保留阶段、实时名次、所选对象和关键解说 | 1倍/2倍、镜头跟随、降低动态；不提供改变赛果的操作 |
| 冲线复盘 | 赛兽停在终点或领奖区，前三与所选对象可见 | 恢复排名、资金、情报验证和分段复盘 | 重看、查看详细数据、返回日程、离开赛场 |

页面状态切换不重建赛事数据，不通过重新打开页面获得第二次封盘或结算机会。

### 3.5 中央逐风兽交互

- 首版仍使用8匹逐风兽，每匹必须拥有独立轮廓、号码、主色、骑师和至少两条可读特征标签。
- 赛前中央展示的是完整赛兽形象和真实待机动作，不使用纯文字名单或相同马形换色代替。
- 鼠标悬停或手柄聚焦时显示名称、号码与一句核心特征；确认选择后使用贝壳金描边、脚下标记和右栏标题三重反馈。
- 点击另一匹逐风兽只切换选择和信息，不扣款、不播放冗长转场。
- 票池支持率高低通过观众旗帜、号码牌热度和文字共同表达，但不能把热门误画成系统推荐。
- 色觉异常或降低动态模式下，仍能通过号码、形状和文字识别八匹逐风兽。

### 3.6 正式观赛表现

- 标准赛事演出目标为25—35秒：起步4—6秒、巡航6—8秒、地形6—8秒、冲刺7—10秒。
- 八兽在同一条横向或轻透视赛道上真实移动，名次变化必须来自超越、失速、路线和冲刺动作，不能只播放排名折线图。
- 镜头优先保持前三与玩家所选逐风兽同屏；距离过大时使用小地图或名次带补充，不让所选对象无故消失。
- 每段最多出现一条主解说，说明可见事实，例如“潮铃在侧风中保持路线”或“白浪提前冲刺后开始掉速”。
- 比赛中不提供能改变结果的按键。欢呼、镜头切换和信息显示只改变表现，确保祝胜券公平。
- 支持1倍、2倍和降低动态；首次正式赛事也允许跳过，但跳过后直接进入同一冲线结果，不改变排名、派彩与时间成本。
- 原有四段排名轨迹保留为赛后次级分析图，不再承担主要观赛画面。

## 4. 逐风兽数据

```yaml
id: cloudfin
name: 云鳍
speed: 78
stamina: 62
burst: 74
stability: 70
course: 82
rider: 68
form: 5
```

| 属性 | 主要影响 |
|---|---|
| 速度 | 巡航与后程基础表现 |
| 耐力 | 巡航、地形和冲刺维持能力 |
| 爆发 | 起步与冲刺 |
| 稳定 | 随机波动与强风适应 |
| 场地 | 地形段与阵雨适应 |
| 骑手 | 四阶段的小幅综合修正 |
| 近况 | 本场整体加减值 |

赛场右栏默认把速度、耐力、爆发和稳定转译为两条优势标签和一条风险标签，同时允许展开精确数值。场地、骑手、精确阶段权重、波动值与天气收益通过造物、NPC消息或公开报道进一步理解。玩家不应被迫同时阅读八张四属性数字表才能完成第一次选择。

## 5. 四段模拟

### 5.1 阶段基础

```text
起步 = 爆发×0.45 + 稳定×0.30 + 骑手×0.25
巡航 = 速度×0.50 + 耐力×0.25 + 场地×0.15 + 骑手×0.10
地形 = 场地×0.35 + 稳定×0.30 + 耐力×0.20 + 骑手×0.15
冲刺 = 爆发×0.40 + 耐力×0.35 + 速度×0.15 + 骑手×0.10
```

### 5.2 天气

- 晴：无额外修正。
- 阵雨：场地属性高于 70 的部分形成正修正，低于 70 形成负修正。
- 强风：稳定属性高于 70 的部分形成正修正，低于 70 形成负修正。

### 5.3 波动

```text
阶段标准差 = 12 - 稳定×0.08
```

每一阶段独立抽取波动并累计。测试和调试必须允许固定随机种子重放。

## 6. 信息层级

### 6.1 默认公开

- 当前天气。
- 八匹逐风兽名称。
- 速度、耐力、爆发、稳定。
- 近况的概括文字。
- 独胜与已解锁票种赔率。
- 玩家当前金贝、投入上限和赛事耗时。

### 6.2 造物辅助信息

首版提供四种永久辅助：

| 造物 | 赛场称呼 | 部署费 | 提供的信息 |
|---|---|---:|---|
| 雨 | 雨势推演 | 4 金贝 | 当前天气对所选逐风兽的修正与全场适应排名 |
| 雷暴 | 雷暴听兆 | 5 金贝 | 所选逐风兽的波动等级、精确标准差和全场波动排名 |
| 水罐 | 补水观察 | 6 金贝 | 所选逐风兽的耐力、稳定与巡航能力比较 |
| 河流 | 河势比照 | 8 金贝 | 所选逐风兽的地形段基础表现与场内排名 |

### 6.3 部署规则

- 只有已经永久发现的造物可以选择。
- 每场最多部署一个。
- 造物不消耗，不从图鉴移除。
- 点击比赛前显示部署费和完整情报。
- 开始比赛时与票面投入一起检查金贝并扣款。
- 免费体验券不免除部署费。
- 辅助不修改随机种子、逐风兽分数、赔率或派彩。
- 玩家可以不使用辅助，免费查看默认公开信息。
- 造物辅助在赛场底部表现为可点击的研读卡，不使用下拉框；选择后在右栏高亮它改变了哪一条判断。
- 情报必须包含来源、适用对象和可验证结论，例如“强风下稳定修正列第2”，不能只显示抽象分数。

## 7. 祝胜券与赔率

### 7.1 首版票种

- 独胜：所选逐风兽获得第一名。
- 入席：所选逐风兽进入前三；账户财富达到 500 金贝后开放。

### 7.2 赔率

赔率以当前模拟公式下的大样本真实命中概率作为安全基准：

```text
参考赔率 = 目标返还率 ÷ 命中概率
目标返还率 = 0.88
```

每场开盘票池由真实概率、少量居民偏好波动和具名NPC票据共同形成。界面显示的数值是“当前预计返还”，玩家购票后仍会有NPC在封盘前追加票。封盘时锁定最终返还。

```text
参考赔率 = 0.88 ÷ 真实命中概率
票池情绪修正 = 预期支持份额 ÷ 当前支持份额
当前预计返还 = 参考赔率 × clamp(票池情绪修正, 0.78, 1.22)
安全上限 = min(20.0, 0.95 ÷ 真实命中概率)
```

赔率始终限制在1.01至安全上限。造物只解释概率来源，不改变开盘票池、NPC追加票、赔率或赛果。

### 7.3 投入限制

```text
单场投入上限 = min(当前金贝, 5000, max(50, 账户财富×10%))
```

- 最低投入 10 金贝。
- 免费体验券使用固定 10 金贝票面。
- 正式投入超过上限时自动限制，并在确认和结算中明确显示。
- 若选择辅助，总可用金贝必须同时覆盖部署费和票面投入。

## 8. 结算

结算顺序：

1. 扣除造物部署费。
2. 使用免费体验券或锁定票面投入。
3. 完成四段模拟。
4. 计算名次与是否命中。
5. 释放锁定本金并发放派彩。
6. 退出赛事`activity_snapshot`并固定推进1潮刻；不叠加比赛演出或回放的现实耗时。
7. 形成一个可解释的财富轨迹节点。

冲线后的同页复盘必须显示：

- 赛事名称、当前预计赔率到封盘赔率的变化。
- 封盘前新增的具名NPC票据。
- 造物名称、部署费和获得的情报。
- 所选逐风兽最终名次。
- 四阶段领先者与所选名次。
- 完整排名。
- 票面投入、赔率、派彩、辅助费和本场净变化。
- 一条“判断与结果”摘要：分别说明赛前依据是否成立、赔率是否发生明显变化、最终失利主要来自哪一赛段或波动；不得把所有未命中都归结为“运气不好”。

赛事历史最多保留最近32场，至少记录赛事编号、日期、天气、冠军、四段八兽完整顺序、玩家票据、封盘赔率、派彩和净变化。晨报读取最近赛果与当前时段赛事，不另建随机新闻副本。

## 9. 与万物合成的闭环

```text
发现雨/雷暴/水罐/河流
→ 永久开放对应竞速观察方式
→ 使用信息判断天气、阶段优势和波动
→ 选择祝胜券与投入
→ 赛后复盘验证判断
→ 获得金贝或新的研究目标
```

后续造物优先扩展新的信息角度，例如拥堵、骑手配合、历史分段和赛道变化。只有在信息玩法稳定后，才考虑能够影响比赛的干预型造物。

## 10. 自动测试与平衡

### 10.1 正确性

- 未发现造物不能出现在辅助列表。
- 部署只扣金贝，不改变发现状态。
- 同一随机种子下，使用与不使用信息造物的排名完全相同。
- 同一赛事的开盘票池、封盘追加票和四段顺序在读档后完全相同。
- 已结算赛事再次提交不会扣款或生成第二份历史。
- 部署费、票面、派彩和最终账户变化严格对应。
- 免费体验券不会免除辅助费。
- 每场只允许一个辅助。
- 赛前页面切换选兽、票种、研读和投入不得提前改变赛事种子、票池或资金。
- “开始赛马”最多触发一次封盘、扣款、模拟、历史写入和固定1潮刻结算。
- 1倍、2倍、降低动态、跳过和重看产生完全相同的排名、派彩与历史记录。

### 10.2 概率

- 每种天气至少模拟 100,000 场校准独胜和前三概率。
- 单个票种的长期理论返还不得超过 0.97。
- 雾步入席等显眼策略不得形成稳定指数增长。
- 改变逐风兽属性、天气公式或随机波动后必须重新生成概率表。

### 10.3 信息价值

试玩记录：

- 各辅助使用率。
- 使用辅助后玩家选择是否改变。
- 辅助费用占平均投入比例。
- 玩家能否在赛后解释输赢。
- 是否出现某个辅助在所有天气和选择中被无脑使用。
- 玩家进入页面5秒内是否先注意到赛兽而不是说明文字。
- 玩家能否不查看公式，说出所选逐风兽的一项优势、一项风险和当前回报是否值得。
- 未命中后，玩家能否区分“判断依据成立但发生波动”和“赛前忽略了明确风险”。

### 10.4 页面与视觉测试

- `1280×720`、`1600×900`、`1920×1080`分别生成赛前、比赛中和冲线复盘快照。
- 三种分辨率下中央八兽、选中标记、当前票据、总支出和开始按钮必须完整可见。
- 键鼠与手柄均可在中央八兽、研读卡、票种、投入和开始按钮间按可预测顺序导航。
- 赛前核心操作区不依赖纵向滚动；完整历史和详细公式允许独立滚动。
- 连续播放、加速、跳过和重看100场不残留赛兽节点、镜头状态、音效或表现锁。

## 11. 原型验收

1. 从世界赛场入口进入独立全屏页面，不打开通用长弹窗。
2. 玩家第一眼能看见中央8匹逐风兽，并能直接点击完成选兽；周围信息始终与选中对象一致。
3. 玩家可在下注前选择一个已发现造物或不使用，四种造物提供不同、明确且可验证的信息。
4. 底部决策条提前显示目标、票种、部署费、票面、总支付能力、预计返还和1潮刻成本。
5. 点击“开始赛马”后在同一页面无缝进入25—35秒正式赛事，能看见实际赛兽、赛道、超越、失速和冲线，而不是只看折线图。
6. 比赛速度和跳过不改变赛果、赔率、资金、历史或时间结算。
7. 冲线后同页复盘四段过程、完整资金变化，以及赛前依据为何成立或失效。
8. 造物使用后永久保留，辅助不会暗中改变赛果与赔率。
9. 每日四场赛事、动态票池、封盘赔率、历史与晨报可以连续跨时段使用并保存。
10. 未看攻略的新玩家能在90秒内完成“看兽—看信息—选票—开始赛马”，并能说出自己的选择理由。

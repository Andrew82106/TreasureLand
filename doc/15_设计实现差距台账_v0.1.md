# 《万物之岛》设计—实现差距台账 v0.1

> 建立日期：2026-07-16  
> 用途：以`00—14`号设计文档为规格，持续记录正式承诺与运行项目之间的差距。  
> 状态定义：`完成`表示已有运行证据和对应测试；`部分`表示存在入口但没有覆盖完整规则；`未实现`表示仍是文档规格。

## 1. 总体结论

当前版本已经形成“地图移动—合成—研究商店—命运牌会—逐风竞速—潜捕鱼市—人物社交”的可玩纵向切片，但距离功能完整产品仍有四组结构性差距：

1. 世界时间底座已支持连续进度、暂停栈、日终停驻、确定性日程、NPC移动和正式存档，但时段边界还需要驱动赛事与新闻的完整刷新。
2. 海岸潜捕与鱼市P0闭环已经替换旧采集点；下一阶段需要装备成长、人物专属订单、住宅陈列和蓝鳍渔业经营连接。
3. 逐风竞速已经形成每日赛事、动态票池、历史、回放与晨报闭环；核心玩法剩余工作集中在正式赛兽动画、音效与更多赛事类型。
4. 长期财富目前只有头衔和曲线，住宅、收藏陈列、商会份契、经营资产与后期目标仍未落地。

## 3. 实施顺序与依赖

### P0：世界状态底座

- 连续世界时钟、暂停栈、日终停驻和统一小数潮刻推进；
- 确定性每日种子、日程快照和边界事件日志；
- 手动存档、日终自动存档、版本迁移与读档测试；
- 地图快速前往、合成、牌会和竞速全部只调用统一时间入口。

没有这一层，NPC位置、鱼价、订单和赛事状态无法可靠复现。

### P1：生活与保底经济

- 用海岸潜捕替换三次点击采集；
- 建立鱼获实例、海生图鉴、鱼市供需、新鲜度、出售预览和订单；
- 将阿葵、米娅、鱼铺经营与晨报接入同一份市场状态。

### P1：人物与世界连接

- 统一NPC目录、时段位置、话题和通用委托；
- 地图显示已认识人物的区域与服务代班；
- 重要牌局、赛事、合成和鱼获写入可产生行为影响的记忆。

### P2：核心玩法深化

- 竞速赛事日程、历史、动态票池、观赛表现和晨报闭环已完成；后续集中到正式赛兽动画、声音和更多赛事内容；
- 牌会加入教学、短/完整牌会、邀请和社交结果；
- 合成首次来源详情、跨模块用途、收藏反馈与万象塔目标链已完成；后续核心玩法深化集中到正式素材、声音与五阶之前的数据审计。

### P3：长期财富产品

- 商会份契、经营摘要、持仓与分红；
- 收藏陈列、生活改善、住宅与财富消耗；
- 最终塔纹、故事节点和可继续游玩的长期目标。

## 4. 每轮完成规则

每次模块开发只有同时满足以下条件，才可以把台账状态向“完成”移动：

1. 设计文档中的数据、流程、边界和失败路径已经进入运行代码；
2. 与其他模块共享同一份状态，不建立临时副本；
3. 玩家可从世界入口实际进入并完成该流程；
4. 根据`12_CI与自动化测试宪法_v0.1.md`补充规则、集成和UI测试；
5. 本地阻断测试、必要视觉快照和远端CI通过；
6. 台账“仍需实现”列删除已经完成的事项，并写明新发现的差距。

## 5. 完整产品验收门

只有当以下证据全部存在时，整个“抹平差距”目标才算完成：

- `00—14`号文档中首版范围和原型验收逐条有运行证据；
- 新游戏到多日循环、存档、读档、日终和长期目标可以连续完成；
- 合成、牌会、竞速、潜捕、鱼市、NPC、地图、商店和后期经济形成双向资源或信息流；
- 不依赖调试按钮、手改存档或控制台才能进入任何正式玩法；
- 所有资金、鱼获、凭证、发现、时间和随机结果满足守恒与确定性测试；
- 三档基线分辨率完成可用性与视觉验收；
- 发布构建和远端CI通过，且不存在未解释的引擎错误或崩溃。

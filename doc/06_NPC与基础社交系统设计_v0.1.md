> 状态：首轮运行规格；统一NPC目录、关系、话题、委托、记忆、日程、地图追踪与环境居民已进入原型
> 上位依据：总文档第 4、10、12、13、17 节  
> 关联文档：`01_万物合成系统设计_v0.1.md`、`02_命运牌会系统设计_v0.1.md`、`04_时间日程与结算设计_v0.1.md`、`07_地图与移动系统设计_v0.1.md`、`08_商店与交易系统设计_v0.1.md`、`14_海岸潜捕与鱼市供需系统设计_v0.1.md`

## 1. 模块定位

NPC负责把合成、牌会、竞速、商店和岛屿信息变成同一个生活世界。玩家不是通过功能菜单选择玩法，而是通过认识人物、听到消息、接受邀请和再次相遇，自然进入各个系统。

首版社交不追求庞大的好感养成，而要保证：

1. 玩家知道下一步可以找谁；
2. NPC在地图上有稳定、可寻找的生活位置；
3. 重要行为会被记住；
4. 信息有明确来源；
5. 社交不会阻断基础玩法。

## 2. 体验目标与非目标

### 2.1 体验目标

- 玩家能从人物语言理解系统，而不是只读教程窗口。
- 同一NPC在牌桌、赛场和日常对话中保持一致性格。
- NPC提供线索和观点，但不提供必赚答案。
- 玩家帮助、欺骗、支持或羞辱人物后，后续对话和邀请发生变化。
- 找人方便，不把社交变成对照时间表的搜索劳动。

### 2.2 非目标

- 首版不制作恋爱、结婚和家庭系统。
- 不以反复送礼作为主要关系成长方式。
- 不要求玩家提高关系才能使用基础商店、低额牌会和常规赛事。
- 不模拟每个路人完整的一生和全天路径。
- 不让高关系NPC凭空提供必赢情报。

## 4. NPC数据结构

```yaml
npc_id: aqiu
name: 阿葵
role: rider
relationship_score: 0
relationship_state: stranger
current_location: wind_coast_stable
schedule_profile: rider_standard
available_topics: []
active_request: null
memories: []
services: [race_info, request_turn_in]
```

核心字段：

- **关系分数**：内部值，范围 -100—100，默认不直接显示；
- **关系状态**：陌生、熟悉、信任、疏远；
- **位置与日程**：当前所在兴趣点及下一次移动时间；
- **话题**：当前可询问的公开信息、人物观点与私人消息；
- **记忆**：玩家造成的重大事件；
- **服务**：商店、牌桌、赛事报名、委托交付等功能入口。

## 5. 基础交互

玩家靠近NPC后显示一个主交互键，进入简洁的对话选项：

| 选项 | 用途 | 时间消耗 |
|---|---|---:|
| 交谈 | 当前状态、环境闲聊、人物反应 | 0固定潮刻，界面暂停 |
| 询问 | 选择一条已知话题，获得事实或观点 | 0固定潮刻，界面暂停 |
| 委托 | 查看、接受或交付人物请求 | 接受/交付0固定潮刻，执行按任务计算 |
| 交易/服务 | 打开商店、牌桌或赛事界面 | 由对应模块决定 |
| 展示/分享造物 | 向NPC展示图鉴中的永久造物或分享其方法 | 0固定潮刻，界面暂停 |
| 展示鱼获/交付订单 | 展示纪录鱼，或向对应收购方交付鱼获 | 展示0固定潮刻，交付由鱼市结算 |
| 深入交谈 | 触发较长人物事件 | 0.5—1潮刻 |

同一NPC的普通闲聊可以重复，但同一条有效信息在刷新前只发放一次，不通过反复点击生成新线索或奖励。

对话文本、选项和人物资料打开时使用`ui_paused`，玩家阅读速度不会改变世界时间。深入交谈和人物事件进入`activity_snapshot`，事件结束后按标注的0.5—1潮刻统一结算，不把对白演出时长再次计入。

## 6. 关系与记忆

### 6.1 关系状态

| 状态 | 内部分数 | NPC表现 |
|---|---:|---|
| 疏远 | -100—-20 | 减少私人话题和邀请，但保留基础公共服务 |
| 陌生 | -19—9 | 礼貌提供公共信息和必要功能 |
| 熟悉 | 10—39 | 增加个人闲谈、普通委托和背景线索 |
| 信任 | 40—100 | 开放私人事件、合作、邀请和更准确的个人信息 |

关系分数只是帮助系统判断状态，不在主界面显示进度条。人物页面通过称呼、最近记忆和关系状态文字表达变化。

### 6.2 关系变化来源

- 完成或违背人物委托；
- 在牌桌上尊重、羞辱、成功诈唬或恶意拖延；
- 长期支持某位骑师或只在热门时接近；
- 公开、分享、献塔或隐瞒与人物相关的造物知识；
- 在人物困难时帮助或趁机获利；
- 对重要对话作出选择。

普通购买、重复闲聊和无脑送礼不会持续增加关系。

### 6.3 记忆结构

首版每名核心NPC保留最近5条重大记忆和2条长期记忆。记忆不仅保存摘要，还必须携带可以被对话、话题、委托、牌桌或赛事读取的行为影响：

```yaml
memory_id: supported_first_race
type: support
importance: 3
created_day: 2
created_tide: 10
summary: 玩家在阿葵不被看好时购买了她的独胜券
relationship_delta: 8
occurrences: 1
effects:
  dialogue_warmth: 1
  private_topic_access: 0
  request_priority: 1
  poker_courage: 0
  race_support: 2
```

新记忆超过上限时，优先保留重要性高或仍影响事件的记录。已经写入长期人物故事的关键记忆不被自动覆盖。

### 6.4 短期更新与长期沉淀

- 同一`memory_id`再次发生时更新出现次数、时间、摘要和最高重要性，不重复结算关系奖励；
- 重要性达到4、被标记为永久事件，或同类记忆出现至少2次时，可以沉淀为长期记忆；
- 长期记忆按“重要性×10 + 出现次数”保留优先级，最多2条；低优先级记录可以退出长期栏，但仍可能保留在最近5条中；
- 同一记忆同时存在于最近与长期栏时，行为影响只计算一次；
- 所有效果总和限制在-20—20，避免旧记忆让人物行为永久失控。

### 6.5 行为影响字段

| 字段 | 首版读取位置 | 正值表现 | 负值表现 |
|---|---|---|---|
| `dialogue_warmth` | 普通交谈、称呼与最近记忆反应 | 更愿意回应和引用共同经历 | 措辞疏离 |
| `private_topic_access` | 私人话题筛选 | 允许因重大经历提前开放特定私人话题 | 不使用 |
| `request_priority` | 委托页与后续人物事件 | 人物更优先提及共同委托 | 降低人物主动提及 |
| `poker_courage` | 同一NPC的牌桌AI | 面对玩家时更敢跟契或施压 | 更谨慎 |
| `race_support` | 阿葵赛事话题与后续邀请 | 承认持续支持与观赛历史 | 认为玩家只追逐热门 |

关系分数负责判断“疏远、陌生、熟悉、信任”大状态；记忆效果负责在同一关系状态内产生具体差异。两者不能互相替代。

## 7. 信息与话题

NPC提供的信息分为三类：

| 类型 | 表现 | 是否保证真实 |
|---|---|---:|
| 公开事实 | 赛程、商店服务、已经发生的结果 | 是 |
| 有来源消息 | 人物观察、记者调查、马厩情况 | 来源可靠时接近事实，但可能不完整 |
| 人物观点 | 对赛兽、牌手、行情或玩家的判断 | 不保证，反映人物立场 |

每条非公开信息必须记录来源、生成时间和有效期。关系提升可以让描述更具体，但不能把观点直接变成系统答案。

鱼群位置、海况判断和未来鱼价同样遵守该规则：渔民可以准确描述自己看见的鱼群，厨师可以真实说明自己的采购需求，但任何人物都不能保证未来时段的市场价格。

## 8. 日程与找人

### 8.1 首版日程原则

- 每名核心NPC每个时段最多在一个主要兴趣点出现；
- 玩家已经认识NPC后，可以在地图人物页查看其当前区域；
- NPC正在参加牌局、比赛或剧情时，地图显示活动状态而不是让其消失；
- 承担必要服务的NPC离开时，由代班人员保留基础功能；
- 重要委托交付对象不应连续两个时段不可找到。

## 9. 委托最小结构

首版委托只承担介绍人物和系统的作用：

```yaml
request_id: aqiu_water_jar_method
giver: aqiu
objective: 在万物图鉴中发现水罐，并向阿葵分享补水方法
source_hint: 陶器已经能固定形状，再想想什么能赋予它用途
time_limit: null
rewards:
  coins: 80
  relationship: 8
  information: next_race_condition
```

规则：

- 普通人物委托不设置紧迫倒计时；
- 限时事件必须在接受前明确说明；
- 委托只读取永久发现状态；分享方法不会移除图鉴中的万物；
- 委托失败不关闭基础功能，只改变人物记忆或错过一次机会。

## 10. 与其他模块的接口

### 10.1 万物合成

- NPC提供万物与组合方向线索，不添加天气、人物或时段等合成条件；
- 造物展示与方法分享只发生在发现完成后，不参与配方判定；
- NPC可以回忆玩家首次发现，但不能改变配方结果。

### 10.2 命运牌会

- 牌桌行为写入同一套人物记忆；
- 牌手性格与日常对话保持一致；
- 牌后闲聊可以产生话题，但不会泄露未公开底牌。

### 10.3 金贝与头衔

- NPC在结算后立即使用玩家当前财富头衔称呼玩家；
- 头衔下降会关闭对应公共资格，但不会删除已经形成的关系和记忆；
- 关系不会替玩家支付买入或绕过票种财富门槛。

### 10.4 商店、地图与时间

- NPC服务入口调用统一商店或玩法界面，不另建一套物品系统；
- 地图展示已认识人物的当前区域与活动状态；
- 世界场景自然走时跨过时段边界时统一更新NPC位置；
- 普通交谈进入`ui_paused`且不追加固定潮刻，深入事件进入`activity_snapshot`并在结束后一次提交时间成本。

### 10.5 海岸潜捕与鱼市

- 公共潜捕入口和蓝鳍鱼铺收购不依赖高关系，相关NPC离开时由代班人员保留服务；
- 老乔发布渔民专单，榕奶奶发布厨师专单，米娅与潮灯收藏家发布收藏家专单；三类订单分别检查数量、尚鲜条件、尺寸与关系门槛；
- NPC消息必须标记来源、生成时间与有效期，不能直接写入隐藏鱼群答案或未来价格；
- 关系提升可以开放更具体的栖息线索、私人订单和人物事件，但不直接提高捕获率或收购价；
- 具名NPC的个人订单进入可追踪交易层；居民、餐饮、渔民、游客和港商的基础供需进入群体账户，二者不得重复计算；
- 背景人物闲谈只复述晨报、船期、订单和市场流水中已经存在的事实，不能凭空制造成交或价格变化；
- 展示纪录鱼只写入人物记忆，出售或订单交付才移除鱼获实例。

## 11. 首版内容范围

- 6 名核心功能NPC；
- 5—8 名环境居民；
- 4种关系状态和最近5条重大记忆；
- 交谈、询问、委托、交易、交付、深入交谈6类交互；
- 每名核心NPC至少 8 条普通对话、3 条状态反应、1 个基础委托；
- 地图人物定位和代班服务；
- 首次合成、教学牌会、首次赛事三段人物引导。
- 首次潜捕公共引导、鱼铺代班服务，以及老乔、榕奶奶、米娅各一类每日专单、交付记忆和一次性关系奖励。

首版不做：恋爱、婚姻、复杂送礼、多人派系、程序化长对话、完整路人生活模拟。

## 12. 原型验收

1. 新玩家在10分钟内通过人物对话找到造化盆、茶摊、商店和赛场。
2. NPC离开常驻点时，玩家仍能通过地图找到其区域，必要服务仍可使用。
3. 同一NPC在牌桌、地图和普通对话中使用同一人物状态与记忆。
4. 重复闲聊不能无限生成金贝、关系或新信息。
5. 玩家做出一项重大行为后，相关NPC下一次见面能给出明确反应。
6. 疏远关系不会关闭基础商店、低额牌会或常规赛事。
7. 疏远关系不会关闭公共潜捕和基础鱼获收购，且任何NPC信息都不能承诺未来必涨鱼价。
8. 风灯庭院以上每天可邀请一名熟悉人物来访；首次关系奖励、30金贝待客费和0.5潮刻只能结算一次。

## 13. 后续待设计

- 核心人物的完整个人故事线；
- 人物之间的关系与事件传播；
- 宴会、私人牌局与多人同时来访；
- 商会控制权和名流社交；
- 专职渔民、鱼铺经营者、餐厅厨师与收藏家的完整人物线；
- 关系状态是否需要更多维度，而不是单一内部数值。

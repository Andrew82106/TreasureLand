# 《万物之岛》商会份契与长期资产系统设计 v0.1

> 状态：首版运行规格
> 上位依据：总文档经济、时间、新闻和后期目标；`03_金贝与基础经济设计_v0.1.md`、`04_时间日程与结算设计_v0.1.md`、`10_原型试玩反馈与全面细化计划_v0.1.md`、`14_海岸潜捕与鱼市供需系统设计_v0.1.md`
> 核心目标：让玩家已经赚到的金贝转化为可持有、会波动、会分红且能从岛上生活理解的长期资产

## 1. 系统定位

商会份契不是现实证券软件的缩小复刻，而是岛民对三项真实产业的共同所有权凭证。玩家必须能够从鱼市、造化盆、逐风赛场、天气和晨报中解释价格为何变化。

系统提供：

- 长期持仓与成本；
- 每日固定报价；
- 白天交易与夜间次日订单；
- 经营利润、价格变化和分红；
- 公开风险提示、晨报与财富轨迹；
- 富商之后的持续财富目标。

系统不提供：

- 逐秒价格跳动；
- 杠杆、卖空、期货与借贷；
- 读取未来随机数的内幕答案；
- 只涨不跌的教学资产；
- 同日买入后立刻卖出的点击套利。

## 2. 三家首版商会

| 份契 | 初始价 | 产业来源 | 主要利好 | 主要风险 |
|---|---:|---|---|---|
| 蓝鳍渔业 | 100金贝 | 鱼获采购、餐厅、居民与加工销售 | 库存足以满足旺盛需求、订单完成、稳定海况 | 强风减产、缺货、玩家与NPC采购成本过高 |
| 万象行运 | 140金贝 | 造物实验后的材料、器具和知识运输 | 新关系、新发现和研究活动增加 | 强风运输成本、研究停滞与重复空跑 |
| 逐风联合会 | 180金贝 | 赛事票务、赛道和服务 | 多场完赛、活跃票池、稳定天气 | 低到场、强风维护和事故成本 |

每家公司固定发行1,000份，玩家首版最多持有600份。持有上限为后续“主要股东/控制人”内容保留空间，不允许通过错误重复购买超过上限。

## 3. 开放与财富口径

- 经营报告和历史报价对所有玩家公开。
- 账户财富首次达到20,000金贝时永久取得商会交易资格；之后即使把现金转换为份契或财富回落，也不会失去入口。
- 份契不计入“账户财富”，因为账户财富继续代表当前金贝、活动中本金和待撮合现金。
- 份契按当前价扣除2%保守变现折价后计入“净资产”。
- 财富轨迹必须把买入、卖出、分红和次日撮合记录为可解释节点。

```text
账户财富 = 当前金贝 + 活动中金贝 + 待撮合买单现金
净资产 = 账户财富 + 鱼获保守变现值 + 份契保守变现值
```

## 4. 交易时间

| 时间 | 行为 |
|---|---|
| 第1—10潮刻 | 使用当日固定报价即时买卖 |
| 第10潮刻后与夜晚 | 休市；只能提交或撤销次日开盘订单 |
| 次日第1潮刻 | 应用昨日经营结果形成的新价格，并撮合隔夜订单 |

同一游戏日每家公司只有一个交易价。时段刷新、打开关闭界面和重复点击不会改变报价。

## 5. 买卖规则

### 5.1 即时买入

- 支付`报价 × 数量 + 1%交易费`，交易费向上取整且至少1金贝。
- 买入形成带日期与单位成本的持仓批次。
- 当日买入的份契在次日才可卖出。

### 5.2 即时卖出

- 只能卖出取得日期早于当前日的可售份契。
- 收到`报价 × 数量 - 1%交易费`。
- 按先进先出移除持仓批次并记录已实现盈亏。

### 5.3 夜间次日订单

- 次日买单按当前价上浮13%冻结最大所需金贝，足以覆盖12%单日涨幅与交易费。
- 次日开盘按新价格成交，多余冻结金贝原路退回。
- 次日卖单只锁定已经可售且未被其他卖单占用的份契。
- 撮合前可以撤单；买单撤销必须全额返还冻结金贝。
- 每个订单拥有唯一编号，最多成交或撤销一次。

## 6. 每日经营结算

### 6.1 蓝鳍渔业

读取：

- 鱼铺库存与重点需求；
- 当日玩家出售金额；
- 已完成鱼市订单；
- 天气造成的捕捞与运输成本。

```text
经营利润 = 批发与加工收入 - 基础运营 - 玩家采购 - 海况附加成本
```

鱼价上涨不自动形成利好。缺货推高鱼价时，蓝鳍也可能因为无货可卖和采购昂贵而利润下降。

### 6.2 万象行运

读取：

- 当日首次发现数量；
- 当日新组合尝试数量；
- 天气造成的运输和损耗成本。

永久万物本身不被出售。行运收入来自发现之后产生的公开研究、器具、记录和运输服务。

### 6.3 逐风联合会

读取：

- 当日完成赛事数量；
- 封盘票池总额；
- 组织方目标返还率；
- 天气与赛道维护成本。

联合会不能读取或修改比赛随机种子。份契价格和赛事赛果共享已经发生的公开结果，但持仓不会改变比赛。

## 7. 价格变化

每家公司在日终生成：

```yaml
revenue: int
costs: int
profit: int
drivers: Array[String]
change_rate: float
next_price: int
```

价格变化由“利润相对该产业常态的偏离 + 当日确定性公开噪声”构成，单日涨跌绝对值不超过12%，价格最低20金贝。相同存档、相同经营输入和相同日种子必须得到完全相同的新报价。

## 8. 分红

- 经营利润为正时，公司以利润的8%形成分红池。
- 每份分红向下取整，且不超过当日价格的2%。
- 当日新买入份契不参与当日分红，至少持有到下一日结算。
- 分红直接进入当前金贝，并形成独立财富轨迹、商会历史和晨报记录。
- 无利润或每份不足1金贝时可以不分红，界面必须明确显示。

## 9. 界面

商会页必须同时展示：

- 当前交易状态与下一边界；
- 当前金贝、待撮合金贝、份契保守变现值；
- 三家公司当前价、昨日变化、经营收入、成本、利润和公开原因；
- 持有数量、可售数量、平均成本、浮动盈亏；
- 即时买卖或次日订单按钮；
- 待撮合订单与撤销入口；
- 最近成交、分红和价格历史；
- “价格可能上涨或下跌、份契不保证回本”的固定风险提示。

入口包括：

- 椰影街“潮汐商会”地标；
- 财富轨迹页；
- 《晴潮晨报》产业栏目。

## 11. 自动测试契约

1. 只有三家固定公司，初始价和公司ID稳定。
2. 同一日同一公司的报价固定，重复打开界面不改变价格。
3. 单日价格变化不得超过12%，价格不得低于20。
4. 交易资格达到一次后永久保留，但风险桌等现金资格仍读取账户财富。
5. 买入扣款、份数、费用与持仓成本原子一致。
6. 当日买入份契不可当日卖出。
7. 卖出按先进先出扣减持仓，所得与费用一致，不得超卖。
8. 隔夜买单冻结金贝计入账户财富但不可重复消费；撤单全额返还。
9. 隔夜订单最多撮合一次，新价低于冻结上限时必须退回差额。
10. 份契保守变现值进入净资产，永久万物仍不计价。
11. 蓝鳍、行运和逐风报告分别读取鱼市、合成和赛事的同源数据。
12. 同一存档和经营输入生成相同报告、价格与分红。
13. 分红只发给满足持有期的份契，不得给当日新买入批次。
14. 版本6存档完整保留持仓、成本、订单、冻结现金、历史和解锁状态。
15. 玩家可以从地图、财富页和晨报进入同一商会状态。

## 12. 首版验收

- 玩家可以解释三家公司各自至少两个涨跌原因。
- 从买入到跨日、价格变化、分红、卖出形成完整闭环。
- 鱼市、合成和竞速的真实活动能够进入经营报告。
- 没有高频点击套利、同日回转、重复订单或无来源分红。
- 份契进入净资产曲线，但不会伪装成当前可花金贝。
- 多日持有即使亏损仍可继续游戏，不会锁死基础恢复活动。

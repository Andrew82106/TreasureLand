# 《万物之岛》商会份契与长期资产系统设计 v0.1

> 状态：首版运行基线；真实持有人、有限现金与深度、集合竞价、公司账、基本价值和现金分红已经接入统一经济状态
> 上位依据：总文档经济、时间、新闻和后期目标；`03_金贝与基础经济设计_v0.1.md`、`04_时间日程与结算设计_v0.1.md`、`14_海岸潜捕与鱼市供需系统设计_v0.1.md`、`18_日终账簿潜捕装备与人物订单设计_v0.1.md`
> 核心目标：让份契成为岛上真实产业的有限所有权，而不是由系统无限承接的涨跌小游戏

## 1. 系统定位

商会份契不是现实证券软件的缩小复刻，而是晴潮岛居民对三项跨岛产业的共同所有权凭证。玩家应能从港口货量、鱼市成交、造物活动、逐风赛事、天气、游客与晨报解释经营和价格变化。

系统必须满足四条底线：

1. 每份份契始终有持有人，不能为承接玩家交易凭空增发或消失；
2. 每笔成交都有真实买卖双方、有限现金和有限可售份数；
3. 商会在玩家不参与时仍有基础业务、收入与成本，玩家只影响世界而不生成整个世界；
4. 分红、回购、经营亏损和交易费用都有明确资金来源与去向。

首版不做杠杆、卖空、连续逐笔撮合、毫秒级盘口和复杂衍生品。

## 2. 三家首版商会

| 份契 | 初始价 | 产业来源 | 主要利好 | 主要风险 |
|---|---:|---|---|---|
| 蓝鳍渔业 | 100金贝 | 鱼获采购、餐饮、加工与群岛出口 | 实际销量、订单完成、稳定海况与出口船到港 | 强风减产、腐损、缺货、采购与运输成本上升 |
| 万象行运 | 140金贝 | 材料、器具、知识与跨岛运输 | 研究活动、外岛订单、新关系与航线恢复 | 强风运输成本、研究停滞、库存积压 |
| 逐风联合会 | 180金贝 | 赛事票务、赛场服务、赞助与来岛客流 | 多场完赛、观众与赞助增长、稳定天气 | 低到场、事故、维护成本与赛事取消 |

每家公司固定发行1,000份，所有账户持仓之和必须恒等于1,000。玩家首版最多持有600份，为后续主要股东与控制权内容保留空间。

## 3. 世界参与者与账户

份契市场不逐个模拟全岛居民，而使用6—10个可审计的集合账户：

| 账户 | 典型行为 |
|---|---|
| 创始人与长期持有人 | 估值稳定，换手低，重视分红与控制权 |
| 本地富商 | 依据产业消息与现金需求调整持仓 |
| 外岛交易者 | 受航班、商船与群岛消息影响，资金进出更明显 |
| 分红持有人 | 偏好盈利稳定、现金充足的商会 |
| 趋势交易者 | 追随成交量、连续涨跌和公开新闻 |
| 恐慌散户 | 在事故、亏损和传闻中集中卖出 |
| 商会回购账户 | 仅在现金和规则允许时回购，不能无限托价 |
| 玩家 | 使用真实金贝提交即时交易或次日订单 |

每个账户至少保存`cash`、`holdings`、`target_holdings`、`risk_preference`、`valuation`、`bid_orders`和`ask_orders`。外岛资金进入或离开必须在世界经济账中标记为外部流入或外部流出。

## 4. 开放与财富口径

- 经营报告、历史报价和基础盘口对所有玩家公开。
- 账户财富首次达到20,000金贝时永久取得商会交易资格。
- 份契不计入“账户财富”，按当前可成交价扣除2%保守折价后计入“净资产”。
- 隔夜买单冻结金贝仍属于账户财富，但不可重复消费。
- 财富轨迹记录买入、卖出、撤单、未完全成交、分红和每日重估。

```text
账户财富 = 当前金贝 + 活动中金贝 + 待撮合买单现金
净资产 = 账户财富 + 鱼获保守变现值 + 份契保守变现值
```

## 5. 交易时间与市场形态

首版采用“每日集合竞价 + 日内有限柜台深度”，兼顾岛屿市场规模与可理解性。

| 时间 | 行为 |
|---|---|
| 次日第1潮刻 | 所有真实账户提交限价单；以最大可成交量形成开盘价并撮合 |
| 第1—10潮刻 | 以当日开盘价显示有限买方深度和卖方深度，玩家可即时吃单 |
| 深度耗尽 | 不再出现无限对手方；玩家只能提交下一日限价单 |
| 第10潮刻后与夜晚 | 休市；可提交或撤销次日订单 |

集合竞价选择“可成交数量最大”的价格；若并列，依次选择未成交差额更小、距参考价值更近、距昨收更近的价格。同一输入、存档种子与订单簿必须得到同一结果。

日内固定的是成交报价，不是无限流动性。界面必须同时显示“当前还能卖出多少”和“当前还能买入多少”。玩家的大额交易会消耗深度，并进入次日各账户的持仓、现金、目标仓位与订单判断。

## 6. 交易与守恒

### 6.1 买入

- 支付`成交价 × 数量 + 1%交易费`，费用向上取整且至少1金贝；
- 对手方减少同量份契并收到成交款；
- 玩家形成带日期与单位成本的持仓批次；
- 当日买入的份契在次日才可卖出。

### 6.2 卖出

- 只能出售取得日期早于当前日的可售份契；
- 玩家收到`成交价 × 数量 - 1%交易费`；
- 对手方支付成交款并增加同量份契；
- 玩家按先进先出移除持仓并记录已实现盈亏。

### 6.3 次日订单

- 买单冻结限价成交额与足额费用，卖单锁定可售份契；
- 订单允许部分成交，未成交部分继续锁定或由玩家撤回；
- 撤单返还未使用现金或份契；订单编号下的每一单位最多成交一次；
- 成交后交易双方现金与持仓同时更新，任一方不足则该部分不得成交。

交易费进入潮汐公所市场维护账户，不能无记录消失。每家公司任意时点都必须满足：

```text
玩家持仓 + 全部集合账户持仓 + 商会库存份额 = 1,000
```

## 7. 商会经营与公司现金

每家公司拥有独立的`company_cash`、生产性资产、债务、留存收益和分红准备金。经营活动由岛上群体账户与外岛部门提供基础流量，玩家行为只作为增量信号。

### 7.1 蓝鳍渔业

只读取鱼市实际业务流水：直接订单成交、鱼铺库存实际销售、加工转化、腐损、采购与运输。不得用日终库存或需求快照伪造销量，也不得把玩家一次卖鱼同时记作“补库存”和“完成需求”。

```text
经营利润 = 实际销售收入 - 鱼获采购 - 加工与运输 - 腐损 - 基础运营
```

### 7.2 万象行运

基础收入来自群岛材料运输、研究机构订单和岛内配送；玩家的新发现、研究购买与高阶实验改变订单结构和声誉，但玩家停做实验不代表整家公司停业。

### 7.3 逐风联合会

基础收入来自观众票务、摊位、赞助与外岛赛事服务；玩家祝胜券只占总票池的一部分。联合会读取已发生的公开赛果和到场数据，不能读取或修改比赛随机种子。

### 7.4 日终现金流

```text
期末公司现金
= 期初公司现金
+ 实际经营现金流入
- 实际经营现金流出
- 已支付分红
- 实际回购支出
```

公司现金、库存和份契账户必须分别守恒；账面利润不能直接当作已经到账的现金。

## 8. 基本价值与价格形成

价格不再由“昨收 × 利润偏差 + 随机噪声”直接生成。商会先形成可解释的基本价值，各账户再围绕不同预期提交订单，由集合竞价决定成交价。

```text
可持续利润 = 最近7日经营利润的指数平滑值
每份基本价值
= max(20,
  (公司现金 + 生产性资产 - 债务 + 可持续利润 × 估值倍数)
  / 1,000)
```

- 经营、天气、订单与事件先改变现金流、资产、风险或可持续利润；
- 账户估值在基本价值附近加入各自风险偏好和公开情绪；
- 买卖不平衡通过真实订单与有限深度影响成交价；
- 单日成交价仍设±12%熔断边界，最低价20金贝；
- 短期情绪可以偏离基本价值，但长期无新信息时应出现回归力量；
- 玩家不交易的长时间模拟中，三家公司仍必须有上涨、下跌和横盘，不能单边衰减或停止变化。

## 9. 分红

- 只有经营盈利、公司现金高于安全运营金且董事会宣布分配时，才把现金转入`dividend_reserve`；
- 分红准备金跨日累积，避免“利润的8%除以1,000后永远为0”；
- 每份整数分红由准备金和合格流通份额计算，支付时真实扣减公司现金与准备金；
- 商会库存份额不领取分红；各持有人按合格份数收到现金；
- 当日新买入份契不参与当日分红；
- 舍入余数留在准备金，不凭空销毁或增发金贝；
- 公司亏损、现金不足或偿付安全线不满足时必须明确显示“不分红”的原因。

## 10. 界面与公开信息

商会页至少显示：

- 当前交易状态、下一边界、昨收、开盘价与±12%边界；
- 当前金贝、冻结金贝、份契保守变现值；
- 1,000份的持有人结构、玩家持仓、可售持仓与商会库存份额；
- 当前买方深度、卖方深度、最近成交量和未成交订单；
- 收入、成本、利润、公司现金、资产、债务、留存收益与分红准备金；
- 基本价值区间及其主要变化原因，不提供“保证上涨”的答案；
- 最近经营事件、成交、分红和价格历史；
- “价格可能上涨或下跌、份契不保证回本”的固定风险提示。

《晴潮晨报》显示游客量、商船/飞艇到港、产业订单、经营摘要和主要盘口变化；地图以船只、货箱、摊位客流和赛场人群提供同源现场观察。

## 12. 自动测试契约

1. 三家公司总发行量固定为1,000，任意交易、读档和迁移后持仓总和不变。
2. 每笔成交都存在资金与份契充足的真实双方；现金和份契同步转移。
3. 日内即时交易不得超过可见深度；深度耗尽后不能生成无限对手方。
4. 集合竞价在相同订单簿下结果确定，优先最大成交量并满足并列规则。
5. 当日买入不可当日卖出；先进先出、费用、部分成交、冻结和撤单严格守恒。
6. 玩家交易会改变对手方持仓、当前深度和次日订单输入。
7. 蓝鳍报告只读取真实采购、成交、加工、腐损与运输流水，不读取库存/需求快照代替销量。
8. 三家公司在玩家连续80日不参与对应玩法时仍产生基础业务；价格具有上涨、下跌、横盘和向基本价值回归的样本。
9. 经营利润、公司现金和账面价值分别计算；亏损与现金不足时不能发放无来源分红。
10. 分红支付前后满足公司现金减少额等于全部持有人现金增加额，舍入余数保留。
11. 单日成交价不超过±12%，最低价不低于20；相同经营、订单和种子生成相同结果。
12. 玩家最大持仓600份、交易资格永久解锁、净资产折价口径保持有效。
13. 晨报、商会页、地图现场和经营报告读取同一组世界经济事实。
14. 存档完整保留账户、订单、冻结资产、深度、公司现金、分红准备金与历史。

## 13. 分阶段落地

### P0：经济真实性

- 固定总发行量与真实集合持有人；
- 有限现金、有限持仓和有限日内深度；
- 每日集合竞价与次日订单；
- 三家公司独立现金账户与非玩家基础经营；
- 玩家交易进入真实对手方与次日价格输入。

### P1：定价与分红

- 基本价值锚、可持续利润和均值回归；
- 公司现金、留存收益、分红准备金与偿付安全线；
- 外岛资金、产业事件与账户差异化策略。

### P2：展示与后期身份

- 持有人结构、盘口深度、未成交订单和现场客流；
- 大宗交易、股东会议、控制权剧情与公司项目；
- 杠杆、卖空和更复杂订单仅在P0/P1守恒稳定后另立规格。

## 14. 首版验收

- 玩家能解释三家公司至少两个经营涨跌原因和一个买卖力量原因；
- 玩家可以完成“次日下单—集合竞价—部分/完全成交—跨日持有—分红或价格变化—有限深度卖出”的闭环；
- 玩家不操作时，鱼业、运输和赛事产业仍持续经营；
- 任何价格变化都有经营、订单、事件或情绪来源，任何金贝与份契都有来源和去向；
- 不存在无限买家、无限卖家、无来源分红和玩家单独驱动全岛经济的问题。

# 角色动画源图生产说明

> 日期：2026-07-15  
> 生成方式：Codex 内置 `imagegen`，参考既有 NPC 肖像、老乔/阿拓图集和美术方向板

## 固定输出契约

### 世界角色

- 源表：8 列 × 4 行，纯色键背景，不画网格、文字、投影或场景。
- 行：down / left / right / up。
- 列：1–4 idle，5–8 四个独立 walk 姿态。
- 运行时：`48×64` 单帧、`8×8` 图集、脚底 `(24, 58)`。
- Idle 仅使用前四列；Walk 的四个独立姿态写入前四列，并确定性复制到后四列以兼容 4/8 帧配置。

### 牌桌角色

- 源表：4 列 × 8 行，纯色键背景，不画桌面、卡牌、筹码、UI、网格或文字。
- 行：idle / talk / think / call / raise / fold / win / lose。
- 每行动四帧，头身中轴不动，只改变表情、手、衣摆与小型手持物。
- 运行时：`96×96` 单帧、`4×8` 图集。

## 确定性后处理

`tools/art/build_character_atlases.py` 负责：

1. 按数学网格切分生成源表；奇数尺寸使用比例边界，不丢最后一行/列。
2. 去除色键与抗锯齿亮边，清理与主体分离的色键碎点。
3. 所有世界帧使用同一整格缩放，不逐帧改变人物大小。
4. 世界帧按完整剪影中心和脚底对齐；牌桌帧按上半身中轴和底部裁切线对齐。
5. 位移若会裁掉非透明像素则立即失败。
6. 输出逐行动画的中轴/底线漂移报告。

复现：

```powershell
python tools/art/build_character_atlases.py
```

完整定量结果见 `doc/其他角色动画图集生成与对齐验证报告_2026-07-15.md`。
